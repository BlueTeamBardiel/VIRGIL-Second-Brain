# WPA3-802.1X

## What it is
Like a corporate building where every employee needs a personal keycard verified by HR — not a shared door code — WPA3-802.1X replaces pre-shared keys with per-user authentication through a RADIUS server. It is the enterprise mode of WPA3, combining WPA3's improved cryptographic handshake (SAE and forward secrecy) with 802.1X's Extensible Authentication Protocol (EAP) framework for individualized credential validation. Each user authenticates uniquely, so revoking one compromised account doesn't affect everyone else.

## Why it matters
In 2017, the KRACK attack exploited weaknesses in WPA2's 4-way handshake — but enterprise deployments using 802.1X with mutual authentication (e.g., EAP-TLS with certificates) were significantly harder to exploit because the attacker couldn't derive session keys without valid credentials. WPA3-Enterprise raises the floor further by mandating 192-bit security mode (CNSA suite) for sensitive environments like government and finance, making brute-force and offline dictionary attacks computationally infeasible.

## Key facts
- WPA3-Enterprise requires a RADIUS authentication server; there is no passphrase to crack offline
- Uses **SAE** (Simultaneous Authentication of Equals) improvements and mandates **forward secrecy**, so captured traffic can't be decrypted later even if long-term keys are compromised
- The **192-bit security mode** uses AES-256-GCMP for encryption and HMAC-SHA-384 for key derivation — required for U.S. government networks
- Supports multiple EAP methods: **EAP-TLS** (certificate-based, strongest), **PEAP**, and **EAP-TTLS**
- Eliminates **PMKID attacks** that plague WPA2-Personal by removing the static PMK concept
