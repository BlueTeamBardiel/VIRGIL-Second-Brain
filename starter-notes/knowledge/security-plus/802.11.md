# 802.11

**IEEE 802.11** is a family of [[wireless networking]] standards that define how devices communicate over radio frequencies in **Wi-Fi networks**. Originally ratified in 1997, the standard has evolved through numerous amendments—including 802.11a/b/g/n/ac/ax—each offering improvements in speed, range, and [[encryption]] capabilities. Understanding 802.11 is fundamental to [[network security]], as wireless networks introduce unique attack surfaces absent from wired infrastructure.

---

## Overview

IEEE 802.11 operates within the **Data Link Layer (Layer 2)** and **Physical Layer (Layer 1)** of the [[OSI Model]], defining both the radio transmission characteristics and the [[MAC (Media Access Control)]] framing structure used to send data wirelessly. The standard is governed by the IEEE (Institute of Electrical and Electronics Engineers) and continuously amended to reflect advances in radio technology, antenna design, and modulation techniques. The Wi-Fi Alliance is a separate industry body that certifies interoperability and branding (e.g., "Wi-Fi 6" corresponds to 802.11ax).

The original 1997 standard supported only 1–2 Mbps using the 2.4 GHz band. Subsequent amendments dramatically increased throughput: 802.11b (11 Mbps), 802.11g (54 Mbps), 802.11n (600 Mbps with MIMO), 802.11ac (Wave 2: ~3.5 Gbps using MU-MIMO and wider channels), and 802.11ax (Wi-Fi 6/6E: ~9.6 Gbps theoretical, with OFDMA for dense environments). Each generation changed modulation schemes, channel widths, frequency bands, and introduced new security mechanisms.

Wireless networks operate in two primary **topologies**: **Infrastructure Mode**, where devices (stations/STAs) connect through a centralized [[Access Point (AP)]], and **Ad-Hoc (IBSS) Mode**, where devices communicate peer-to-peer without an AP. Most enterprise and home deployments use infrastructure mode, often with multiple APs forming an **Extended Service Set (ESS)**—commonly called a wireless LAN (WLAN). The AP broadcasts a **SSID (Service Set Identifier)** that identifies the network to clients.

Radio frequency allocation is critical: the **2.4 GHz band** offers three non-overlapping channels (1, 6, 11 in North America) and better range but suffers more interference from Bluetooth, microwaves, and neighboring networks. The **5 GHz band** provides up to 24 non-overlapping channels and faster speeds but shorter range. **6 GHz (Wi-Fi 6E)** adds 59 additional 20 MHz channels but requires supporting hardware and is only available in newer devices.

The **CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance)** mechanism governs how 802.11 devices share the wireless medium. Unlike Ethernet's CSMA/CD (Collision Detection), wireless devices cannot simultaneously transmit and listen, so they use backoff algorithms and optional **RTS/CTS (Request to Send/Clear to Send)** handshakes to reduce collisions. This fundamental difference makes wireless far more susceptible to interference, eavesdropping, and denial-of-service attacks than wired equivalents.

## How It Works

### Frame Types
802.11 defines three categories of frames, each serving a distinct function:

- **Management Frames**: Handle connection setup and teardown. Examples include Beacon, Probe Request/Response, Authentication, Association Request/Response, and Deauthentication frames. These are typically **unencrypted** in most deployments, creating significant security vulnerabilities.
- **Control Frames**: Manage access to the medium. Examples: ACK, RTS, CTS, Block ACK.
- **Data Frames**: Carry actual payload data (encapsulated upper-layer protocols).

### Connection Process (Step-by-Step)

**1. Scanning**
A client scans for networks either passively (listening for Beacon frames) or actively (sending Probe Request frames):
```
[Client] --Probe Request (SSID or wildcard)--> [AP]
[AP]     --Probe Response (capabilities)------> [Client]
```
Beacon frames are broadcast by APs every ~102.4ms and contain: SSID, BSSID, supported rates, channel, RSN (security) information.

**2. Authentication**
Open System Authentication (the default modern method) is a simple two-frame exchange:
```
[Client] --Authentication Request (Seq 1)---> [AP]
[AP]     --Authentication Response (Seq 2)--> [Client]
```
Legacy **Shared Key Authentication** (WEP-era) used a challenge-response with RC4—now deprecated and broken.

**3. Association**
After authentication, the client associates with the AP, negotiating capabilities:
```
[Client] --Association Request (SSID, rates, HT/VHT caps)--> [AP]
[AP]     --Association Response (AID assigned)-------------> [Client]
```
The AP assigns an **Association ID (AID)** for power-saving purposes.

**4. 4-Way Handshake (WPA2/WPA3)**
After 802.1X or PSK authentication, a four-message exchange derives session keys:
```
AP   --> Client : EAPOL-Key (ANonce)
Client --> AP   : EAPOL-Key (SNonce, MIC)
AP   --> Client : EAPOL-Key (GTK, MIC)
Client --> AP   : EAPOL-Key (ACK)
```
This derives the **PTK (Pairwise Transient Key)** for unicast encryption and installs the **GTK (Group Temporal Key)** for broadcast/multicast.

**5. Data Transfer**
Data frames are encrypted using CCMP (WPA2) or GCMP (WPA3) and transmitted using CSMA/CA:
```
1. Sense medium → if idle, wait DIFS (Distributed IFS)
2. If still idle, transmit
3. If busy, apply random backoff (contention window)
4. Receive ACK confirmation
```

### Useful Commands for Wireless Analysis

**Linux – View wireless interfaces and capabilities:**
```bash
iw list                         # Full capability dump
iw dev wlan0 info               # Interface details
iw dev wlan0 scan               # Active scan for APs
iwconfig wlan0                  # Legacy wireless config
```

**Monitor Mode (for packet capture):**
```bash
sudo ip link set wlan0 down
sudo iw dev wlan0 set type monitor
sudo ip link set wlan0 up
sudo iw dev wlan0 set channel 6
```

**Capture 802.11 frames with tcpdump:**
```bash
sudo tcpdump -i wlan0mon -w wireless_capture.pcap type mgt
```

**Aircrack-ng suite – Scan and capture:**
```bash
sudo airmon-ng start wlan0          # Enable monitor mode
sudo airodump-ng wlan0mon           # Scan all channels
sudo airodump-ng -c 6 --bssid AA:BB:CC:DD:EE:FF -w cap wlan0mon
```

## Key Concepts

- **SSID (Service Set Identifier)**: The human-readable name of a wireless network, up to 32 bytes. Can be hidden (not broadcast in Beacon frames) but this provides no real security since SSIDs appear in Probe Requests and Association frames.
- **BSSID (Basic Service Set Identifier)**: The MAC address of the AP radio interface. Uniquely identifies each AP in an environment. Used to distinguish networks with the same SSID.
- **WPA2 (Wi-Fi Protected Access 2)**: Security protocol using AES-CCMP for encryption and IEEE 802.1X or PSK for authentication. Replaced WEP and WPA (TKIP). Vulnerable to offline dictionary attacks against the 4-way handshake and KRACK attacks (CVE-2017-13077 through CVE-2017-13088).
- **WPA3**: The current generation wireless security standard, introducing **SAE (Simultaneous Authentication of Equals)** to replace PSK, providing **forward secrecy** and resistance to offline dictionary attacks. Also mandates PMF (Protected Management Frames).
- **MIMO (Multiple Input, Multiple Output)**: Antenna technology using multiple transmit and receive antennas simultaneously to increase throughput. **MU-MIMO** (Multi-User MIMO) allows simultaneous transmission to multiple clients, introduced in 802.11ac Wave 2.
- **OFDMA (Orthogonal Frequency Division Multiple Access)**: Introduced in 802.11ax, allows an AP to divide a channel into smaller sub-channels (resource units) to serve multiple clients simultaneously, dramatically improving efficiency in dense environments.
- **PMF (Protected Management Frames) / 802.11w**: An amendment that encrypts and authenticates management frames (specifically deauthentication and disassociation) to prevent spoofed disconnection attacks. Mandatory in WPA3, optional in WPA2.
- **Beacon Frame**: Management frame broadcast by an AP approximately every 100ms, advertising the network's existence, SSID, supported data rates, security capabilities (RSN IE), and timing synchronization information.
- **Hidden SSID**: Configuration where the AP omits the SSID from Beacon frames. Provides security through obscurity only—SSID is revealed during association and can be captured passively.

## Exam Relevance

**SY0-701 Domain Mapping**: 802.11 appears primarily in **Domain 2 (Threats, Vulnerabilities, and Mitigations)** and **Domain 3 (Security Architecture)**, specifically in wireless security, infrastructure hardening, and attack recognition topics.

**High-Frequency Exam Topics:**
- Know the **encryption protocol progression**: WEP (RC4, broken) → WPA (TKIP, deprecated) → WPA2 (AES-CCMP) → WPA3 (AES-GCMP with SAE). Security+ frequently tests which protocol to recommend and which to avoid.
- **Evil Twin AP** attacks: A rogue AP with the same SSID as a legitimate network. Clients may automatically connect, enabling MitM. Paired with deauthentication attacks to force reconnection.
- **Deauthentication attacks**: Exploit unprotected management frames to disconnect clients—**802.11w/PMF** is the countermeasure. Know that WPA3 mandates PMF.
- **WPS (Wi-Fi Protected Setup)**: PIN-based vulnerability (Pixie Dust attack, brute-force against 8-digit PIN split into two 4-digit halves). The exam expects you to know WPS should be disabled.
- **War driving/War chalking**: Discovering and mapping wireless networks. Tools: Kismet, NetStumbler.
- **Captive portals**: Not a security control—they operate at Layer 7 and do not encrypt traffic. Do not confuse them with WPA2.
- **CCMP vs. TKIP**: CCMP = AES-based, strong; TKIP = RC4-based wrapper, weak. WPA2 with TKIP (WPA2 Mixed Mode) is weaker than WPA2 with CCMP only.
- **Disassociation vs. Deauthentication**: Both can be spoofed without PMF. Deauthentication forces a full reauthentication; disassociation requires only reassociation.

**Common Gotchas:**
- WPA3 does NOT replace all WPA2 vulnerabilities retroactively—**transition mode** (WPA2/WPA3 coexistence) can be downgrade-attacked.
- "Wi-Fi 5" = 802.11ac, "Wi-Fi 6" = 802.11ax, "Wi-Fi 6E" = 802.11ax with 6 GHz band.
- The 4-way handshake occurs after authentication—capturing it offline enables dictionary attacks against WPA2-PSK.

## Security Implications

### WEP (Broken – Do Not Use)
**Wired Equivalent Privacy** used RC4 with a 24-bit Initialization Vector (IV). IVs repeat, enabling statistical recovery of the key using tools like `aircrack-ng` with as few as 40,000 captured packets. Completely deprecated since 2004 and removed from modern devices.

### WPA2 KRACK Attack (CVE-2017-13077 to CVE-2017-13088)
**Key Reinstallation Attack** exploited the 4-way handshake by replaying cryptographic handshake messages, forcing nonce reuse in CCMP. An attacker in range could decrypt (and in some cases forge) traffic. Patched via OS updates in late 2017. Highlights the risk of key management vulnerabilities rather than algorithm weakness.

### PMKID Attack (2018)
Discovered by Jens Steube (hashcat author): An attacker can capture the **PMKID** (Pairwise Master Key Identifier) from a single EAPOL frame without capturing the full 4-way handshake—or even waiting for a client to connect. The PMKID can be brute-forced offline:
```
PMKID = HMAC-SHA1-128(PMK, "PMK Name" || AP_MAC || Client_MAC)
```
This dramatically lowers the bar for offline WPA2-PSK cracking.

### Evil Twin / Rogue AP Attacks
An attacker creates an AP with identical SSID and stronger signal. Clients auto-connect (especially after deauthentication from legitimate AP). Tools: `hostapd-wpe` (captures MSCHAP credentials from 802.1X attempts), `bettercap`. Detection: Wireless Intrusion Prevention Systems (WIPS) monitoring for BSSID anomalies.

### Deauthentication / Disassociation Flood (DoS)
Since management frames are unauthenticated in WPA2, attackers can spoof deauthentication frames from the AP's BSSID to disconnect all clients:
```bash
# Targeted deauth (aireplay-ng)
sudo aireplay-ng --deauth 100 -a <AP_BSSID> -c <Client_MAC> wlan0mon
```
Countermeasure: 802.11w PMF (mandatory in WPA3).

### WPS Pixie Dust Attack
Exploits weak randomness in the WPS PIN exchange used by many consumer routers. Tools: `reaver`, `bully`. Success rate is high against vulnerable firmware. Many routers are still vulnerable if WPS is not manually disabled.

### Bluetooth/IoT Interference and Crosstalk
Devices operating in the 2.4 GHz band can interfere with 802.11b/g/n networks. While not a direct attack vector, heavy interference can be used as a primitive DoS.

## Defensive Measures

### Protocol Selection
- **Always deploy WPA3-Personal (SAE)** or at minimum **WPA2-Personal with AES-CCMP only**. Explicitly disable TKIP, WEP, and WPA2 Mixed Mode.
- Enable **WPA3-Enterprise** with 802.1X and EAP-TLS (certificate-based) for enterprise environments.
- Disable **WPS** on all access points without exception.
