# 4G

**4G** (Fourth Generation) is the fourth major generation of cellular network technology, defined by the **International Telecommunication Union (ITU)** under the **IMT-Advanced** standard, superseding [[3G]] and preceding [[5G]]. The dominant implementation is **Long-Term Evolution (LTE)**, a packet-switched radio access technology delivering theoretical downlink speeds of 100 Mbps to 1 Gbps. 4G fundamentally shifted mobile networking by eliminating circuit-switched voice infrastructure in favor of an all-IP architecture, making it a critical attack surface in modern [[Mobile Security]] and enterprise network design.

---

## Overview

4G LTE was standardized by the **3rd Generation Partnership Project (3GPP)** in Release 8 (2008), with commercial deployments beginning in 2009 in Scandinavia by TeliaSonera. The architecture replaced the traditional Base Transceiver Station (BTS) and Radio Network Controller (RNC) model with a flatter, simplified two-node structure: the **eNodeB** (Evolved Node B) handles radio access functions, and the **Evolved Packet Core (EPC)** manages session control, authentication, and IP routing. This eliminated the need for a dedicated circuit-switched core network entirely — voice calls are handled via **VoLTE (Voice over LTE)**, treating voice as just another IP data stream.

The **Evolved Packet Core** consists of several functional elements: the **Mobility Management Entity (MME)** handles control-plane signaling, subscriber tracking, and authentication; the **Serving Gateway (S-GW)** routes user-plane data packets; the **PDN Gateway (P-GW)** provides the interface to external IP networks (the public internet or private APNs); and the **Home Subscriber Server (HSS)** acts as a central authentication and subscription database, functionally analogous to the [[RADIUS]] server in Wi-Fi architectures. The separation of control-plane and user-plane traffic in this architecture was a deliberate design decision enabling more efficient scaling.

Radio access in LTE uses **Orthogonal Frequency Division Multiple Access (OFDMA)** for the downlink and **Single Carrier FDMA (SC-FDMA)** for the uplink. These techniques divide the available spectrum into narrow orthogonal subcarriers spaced 15 kHz apart, packed into **Resource Blocks (RBs)** of 12 subcarriers each spanning 180 kHz. **MIMO (Multiple Input, Multiple Output)** antenna techniques — commonly 2×2 or 4×4 configurations — are used to increase throughput by transmitting multiple spatial data streams simultaneously. LTE-Advanced (3GPP Release 10) further enhanced this with **Carrier Aggregation**, allowing devices to bond up to five component carriers across different frequency bands.

4G operates across a wide range of frequency bands globally. In the United States, major bands include Band 2 (1900 MHz PCS), Band 4 (1700/2100 MHz AWS), Band 12 (700 MHz), and Band 66 (1700/2100 MHz extended AWS). Low-band spectrum (below 1 GHz) provides superior coverage and building penetration; high-band spectrum (above 2.5 GHz) provides higher throughput with shorter range. **Band 48 (CBRS, 3.5 GHz)** enables private LTE deployments — a significant homelab and enterprise capability allowing organizations to run their own 4G networks using software-defined radio infrastructure.

The security model of 4G represented a substantial improvement over 2G/3G, introducing **mutual authentication** between the device and network (preventing fake base station impersonation during authentication), **mandatory encryption** of the radio interface, and **integrity protection** of control-plane signaling. However, the all-IP architecture also dramatically expanded the attack surface, exposing the EPC to IP-based threats and making protocols like **Diameter** (which replaced SS7 for 4G inter-carrier signaling) a new and heavily exploited attack vector.

## How It Works

### Connection Establishment and Authentication

The LTE connection process is a multi-phase procedure involving cryptographic mutual authentication, radio resource negotiation, and IP address assignment.

**Step 1 — Cell Search and Synchronization**
The UE (User Equipment, i.e., the mobile device) scans for cells by detecting Primary and Secondary Synchronization Signals (PSS/SSS) broadcast by eNodeBs. The UE reads the **Master Information Block (MIB)** and **System Information Blocks (SIBs)** from the Physical Broadcast Channel (PBCH) to learn cell parameters (bandwidth, PLMN identity, TAC).

**Step 2 — Random Access (RACH Procedure)**
The UE transmits a random preamble on the Physical Random Access Channel (PRACH). The eNodeB responds with a **Random Access Response (RAR)** allocating temporary resources. The UE then sends an **RRC Connection Request**.

**Step 3 — EPS AKA (Authentication and Key Agreement)**
This is the core security mechanism, defined in 3GPP TS 33.401.

```
UE  <----->  MME  <----->  HSS

1. UE sends IMSI (or GUTI if previously attached) to MME
2. MME requests Authentication Vectors from HSS
3. HSS generates AV = {RAND, AUTN, XRES, KASME} using Ki stored on SIM
   - Ki: 128-bit secret key shared between SIM and HSS
   - AUTN: Network Authentication Token (proves network identity to UE)
   - XRES: Expected Response (used to verify UE)
   - KASME: Root key for deriving session keys
4. MME sends RAND + AUTN to UE
5. UE's SIM verifies AUTN (mutual auth), computes RES using Ki
6. UE returns RES → MME compares RES to XRES
7. If match: authentication successful
8. Both sides derive: KeNB, KNASenc, KNASint from KASME
```

**Step 4 — Security Mode Command**
The MME sends a Security Mode Command specifying the negotiated encryption algorithm (EEA0=null, EEA1=SNOW 3G, EEA2=AES-CTR, EEA3=ZUC) and integrity algorithm (EIA1=SNOW 3G, EIA2=AES-CMAC, EIA3=ZUC). The RRC and NAS layers are now integrity-protected and encrypted.

**Step 5 — Default Bearer Establishment**
The MME signals the S-GW and P-GW to establish a **GTP (GPRS Tunneling Protocol)** tunnel. GTP-U (User Plane) uses **UDP port 2152** to encapsulate user data between network nodes. GTP-C (Control Plane) uses **UDP port 2123** for tunnel management signaling.

**Step 6 — IP Address Assignment**
The P-GW assigns the UE an IP address (IPv4/IPv6) via the PDN connection. The UE can now send/receive internet traffic, which flows: UE → (LTE-Uu radio) → eNodeB → (S1-U GTP tunnel) → S-GW → (S5/S8 GTP tunnel) → P-GW → Internet.

## Key Concepts

- **IMSI (International Mobile Subscriber Identity)**: A 15-digit globally unique identifier stored on the SIM card, structured as MCC (3 digits) + MNC (2-3 digits) + MSIN. The IMSI is transmitted in plaintext during initial attach before a GUTI is assigned, making it harvestable by IMSI catchers ([[IMSI Catcher]]).

- **GUTI (Globally Unique Temporary Identifier)**: A temporary identifier assigned by the network after initial authentication to avoid exposing the IMSI in subsequent connections. Structured as PLMN + MMEGI + MMEC + M-TMSI. Rotated periodically by the network.

- **eNodeB (Evolved Node B)**: The 4G base station that handles all radio access layer functions, including scheduling, HARQ retransmission, header compression (ROHC), and ciphering at the PDCP layer. Unlike 3G RNC architecture, eNodeBs communicate directly with each other via the X2 interface for handover coordination.

- **EPS-AKA (Evolved Packet System Authentication and Key Agreement)**: The mutual authentication protocol for LTE, defined in 3GPP TS 33.401. Provides network-to-device authentication (via AUTN verification), prevents IMSI catchers from successfully completing authentication, and derives a hierarchy of session keys (KASME → KNASenc, KNASint, KeNB → KUPenc, KRRCenc, KRRCint).

- **Diameter Protocol**: The successor to [[SS7]] RADIUS for 4G inter-network signaling, used on interfaces S6a (MME-HSS), Gx (PGW-PCRF), and Rx. Operates over TCP/SCTP port 3868. Despite being newer than SS7, Diameter has its own significant attack surface including spoofing, location tracking, and intercept attacks, as documented by researchers at P1 Security and Positive Technologies.

- **APN (Access Point Name)**: A configurable identifier that routes a device's data connection to a specific PDN gateway and external network. Enterprise deployments use private APNs to route mobile device traffic through corporate infrastructure — functionally similar to a VPN but at the carrier level.

- **Carrier Aggregation (CA)**: An LTE-Advanced feature allowing a device to simultaneously use multiple frequency carriers (up to 5 in Release 10, up to 32 in Release 13) to aggregate bandwidth. Carriers can be contiguous, non-contiguous intra-band, or inter-band (e.g., combining Band 12 and Band 4 simultaneously).

## Security Implications

### Residual IMSI Exposure
Despite the GUTI mechanism, IMSI exposure remains possible. Network paging messages broadcast IMSI in some implementations; certain handover and attach procedures force IMSI retransmission. **IMSI catchers** (Stingrays, DRT boxes) exploit this by broadcasting a stronger signal than legitimate towers, forcing devices to attach and reveal their IMSI before the catcher drops the connection.

### Diameter Protocol Attacks
The Diameter protocol, while more modern than SS7, shares many of the same conceptual vulnerabilities. **Positive Technologies research (2018)** demonstrated that 100% of tested 4G networks were vulnerable to subscriber location tracking via Diameter, and 77% were vulnerable to fraudulent service use. Key attack classes include:

- **Location Tracking**: Sending `Insert-Subscriber-Data` or `Cancel-Location` requests to the HSS to force re-registration, leaking subscriber location.
- **Intercept Facilitation**: Triggering re-authentication to force downgrade or redirect calls.
- **Denial of Service**: Cancelling a subscriber's location registration, disconnecting them from service.

### LTE Protocol-Layer Attacks
Academic research has revealed multiple design-level vulnerabilities:

- **aLTEr Attack (2019)**: Exploits the lack of integrity protection on LTE user-plane (GTP-U) data. An attacker with radio access can perform DNS spoofing by bit-flipping encrypted UDP packets. Published by Rupprecht et al. at IEEE S&P 2019 — CVE not assigned (design flaw, not implementation bug).
- **ReVoLTE (2020)**: Exploits keystream reuse in VoLTE implementations. LTE spec requires fresh keys for each call, but some implementations reuse the keystream. An attacker can XOR-recover plaintext of a victim's call by making a second call immediately after. Demonstrated against German carriers.
- **LTE Rogue Base Station**: Unlike 2G where no mutual auth exists, LTE requires AUTN verification; however, a rogue BS can still perform downgrade attacks (forcing the UE to fall back to 2G/3G) or denial-of-service by sending spoofed RRC release messages (no integrity protection on pre-authentication messages).

### SIM Swapping
Not a 4G-specific flaw but heavily exploits 4G carrier authentication procedures. Attackers socially engineer carrier customer service to transfer a victim's phone number to an attacker-controlled SIM, intercepting SMS-based [[MFA]] and VoLTE calls. High-profile cases include the 2020 Twitter hack (facilitated by SIM swaps) and numerous cryptocurrency theft cases.

### GTP Exploitation
Attackers with access to GTP infrastructure (S1-U, S5/S8 interfaces) can:
- Create unauthorized GTP tunnels to access internal carrier networks.
- Hijack existing GTP sessions by spoofing TEIDs (Tunnel Endpoint Identifiers).
- Use carriers as pivots to reach backend OA&M (Operations, Administration, and Maintenance) networks.

**Shodan regularly indexes exposed GTP ports (UDP 2152, 2123)** on misconfigured carrier equipment.

## Defensive Measures

### For End Users and Enterprise Administrators

**1. Disable 2G/3G fallback where possible**
Modern Android devices (Android 12+) and iOS 16+ support "LTE Only" or "5G/LTE Only" modes, preventing downgrade attacks.

```
Android: Settings → Network → Preferred Network Type → LTE Only
iOS: Settings → Cellular → Cellular Data Options → Voice & Data → LTE
```

**2. Deploy Private APN with certificate pinning**
Enterprise MDM platforms (Jamf, Microsoft Intune, VMware Workspace ONE) can provision private APNs that route all mobile traffic through a corporate security stack (NGFW, DLP, proxy).

**3. Use end-to-end encrypted communications**
Since LTE encryption terminates at the eNodeB (not end-to-end), assume radio-layer confidentiality only extends to the carrier. Deploy application-layer encryption ([[TLS 1.3]], [[Signal Protocol]]) for sensitive communications.

**4. SIM Security Hardening**
- Enable SIM PIN (prevents SIM removal/reuse).
- Register a carrier account PIN/passphrase to defend against social engineering SIM swaps.
- Use hardware security keys (FIDO2) rather than SMS for MFA wherever possible.

**5. Monitor for IMSI Catcher Activity**
Applications such as **AIMSICD (Android IMSI Catcher Detector)** and **SnoopSnitch** analyze radio environment parameters (cell tower changes, signal anomalies, cipher algorithm downgrades) to detect potential catcher activity.
