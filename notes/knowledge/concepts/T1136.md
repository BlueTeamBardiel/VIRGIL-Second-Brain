# T1136

## What it is
Like a burglar who copies a house key and hides it in a flower pot for later visits, attackers create new user accounts to maintain persistent, legitimate-looking access to a compromised system. T1136 (Create Account) is the MITRE ATT&CK technique where adversaries add local, domain, or cloud accounts to sustain access without relying on malware or exploits.

## Why it matters
During the 2020 SolarWinds campaign, threat actors created new accounts in Azure Active Directory after gaining initial access, blending in with legitimate administrative accounts to persist undetected for months. Defenders who monitor for unexpected account creation events (Windows Event ID 4720 for local accounts, 4741 for domain accounts) can catch this technique early before the attacker establishes deeper footholds.

## Key facts
- **Sub-techniques**: T1136.001 (Local Account), T1136.002 (Domain Account), T1136.003 (Cloud Account) — each requires different detection strategies
- **Windows Event IDs to monitor**: 4720 (local account created), 4741 (computer/domain account created), and 4728/4732 for subsequent group additions
- **Privilege requirement**: Creating domain accounts typically requires Domain Admin privileges, making it a post-privilege-escalation action
- **Cloud detection**: In AWS/Azure/GCP, monitor IAM CreateUser, AddUser, or similar API calls via CloudTrail or Azure Activity Logs
- **Common attacker move**: Naming rogue accounts to mimic service accounts (e.g., `svc_backup`, `admin_temp`) to evade casual review
