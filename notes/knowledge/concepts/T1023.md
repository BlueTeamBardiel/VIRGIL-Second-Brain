# T1023

## What it is
Like leaving a trail of breadcrumbs from your front door to your treasure chest, shortcut files (.lnk) create a visible path directly to executables or scripts an attacker wants users to click. T1023 describes adversaries placing malicious Windows Shell Link (.lnk) shortcut files in user-accessible locations to establish persistence or achieve execution by tricking users into launching them.

## Why it matters
The Emotet malware campaign extensively used malicious .lnk files delivered via phishing emails — when a victim double-clicked what appeared to be an invoice shortcut, the .lnk silently executed a PowerShell command to download the payload. Defenders monitoring Startup folders and Desktop locations for unexpected .lnk files pointing to unusual executables (like those in %TEMP% or %APPDATA%) can catch this technique early before the payload executes.

## Key facts
- .lnk files can embed command-line arguments, meaning a shortcut appearing to open "Invoice.pdf" can actually run `powershell.exe -enc [base64payload]`
- Common persistence locations include `C:\Users\<user>\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup` and the Desktop
- .lnk files have a distinctive file signature (magic bytes `4C 00 00 00`) useful for forensic identification even when renamed
- Attackers frequently spoof icons to match legitimate applications (Word, Excel, PDF reader) to lower victim suspicion
- Detection strategy: audit .lnk files where the target path points outside expected program directories (e.g., System32, Program Files)
