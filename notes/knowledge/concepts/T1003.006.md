# T1003.006

## What it is
Like a thief who doesn't pick the lock but instead duplicates the master key from a wax impression, DCSync tricks Active Directory into *willingly handing over* password hashes. It is a credential dumping technique where an attacker impersonates a Domain Controller and uses the Directory Replication Service (DRS) protocol to request password data — including NTLM hashes and Kerberos keys — directly from a legitimate DC.

## Why it matters
In the 2020 SolarWinds breach, attackers used DCSync after achieving sufficient AD privileges to silently extract credential material for dozens of accounts without ever touching LSASS memory on the DC itself. Defenders who only monitor for local memory access tools like Mimikatz will completely miss this attack vector, which is why network-level detection of unexpected replication traffic is critical.

## Key facts
- Requires **Replicating Directory Changes** and **Replicating Directory Changes All** permissions — privileges normally held only by DCs and certain admin accounts
- Executed via Mimikatz with the command: `lsadump::dcsync /domain:corp.local /user:Administrator`
- Does **not** require interactive logon or code execution on the DC itself — purely network-based
- Generates Windows Event ID **4662** (An operation was performed on an object) — the primary detection artifact
- Often used to pull the **KRBTGT** hash, enabling Golden Ticket attacks for persistent domain access
