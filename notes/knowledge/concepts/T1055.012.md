# T1055.012

## What it is
Like a puppeteer slipping their hand into a marionette, an attacker writes malicious code directly into the memory space of a legitimate, already-running process. Process Hollowing is a technique where an attacker spawns a legitimate process in a suspended state, carves out its original executable code, and replaces it with malicious payload — all while the process retains its trusted identity.

## Why it matters
Dridex banking malware famously used process hollowing to inject itself into `svchost.exe`, inheriting that process's trusted reputation to exfiltrate banking credentials while evading signature-based AV tools that saw only a legitimate Windows service. Defenders catching this look for mismatches between a process's image on disk and what's actually executing in memory — a tell-tale sign of hollowing.

## Key facts
- The target process is created with `CREATE_SUSPENDED` flag, preventing it from executing before the swap occurs
- Attacker uses `ZwUnmapViewOfSection` (or `NtUnmapViewOfSection`) to hollow out the legitimate code from memory
- Malicious code is written in using `WriteProcessMemory`, and execution resumes via `SetThreadContext` + `ResumeThread`
- The process appears legitimate in Task Manager and standard process listings because the PID and name belong to a trusted binary
- Detection relies on memory forensics tools (like Volatility) comparing the process's VAD entries or memory-mapped sections against the legitimate on-disk executable hash
