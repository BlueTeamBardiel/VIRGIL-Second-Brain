# T1059.003 - Command-Shell

## What it is
Think of `cmd.exe` as a master key that was already hanging on the wall — attackers don't need to pick the lock, they just use the key that's already there. T1059.003 refers to adversary abuse of the Windows Command Shell (`cmd.exe`) to execute commands, scripts, and binaries as part of their attack chain. It's a Living-off-the-Land (LotL) technique because it leverages a trusted, built-in OS component.

## Why it matters
During the 2020 SolarWinds compromise, attackers used `cmd.exe` to execute reconnaissance commands and invoke child processes after establishing a foothold via the SUNBURST backdoor — blending into normal administrative traffic. Defenders monitoring for anomalous `cmd.exe` parent-child process relationships (e.g., a web server spawning `cmd.exe`) would have flagged this activity early.

## Key facts
- **Parent process anomalies are a primary detection signal** — `cmd.exe` spawned by `iis.exe`, `outlook.exe`, or `msiexec.exe` is almost always suspicious
- **Encoded or chained commands** using `cmd.exe /c` with `&&` or `|` operators are commonly used to execute multi-stage payloads in a single line
- **Script block logging and Process Creation auditing** (Windows Event ID 4688) are the core logging controls used to detect this technique
- **Disabling command shell via AppLocker or Software Restriction Policies** is a valid mitigation but must be tested carefully to avoid breaking legitimate admin workflows
- **Maps to MITRE ATT&CK Tactic: Execution** — commonly chained with T1059.001 (PowerShell) and T1047 (WMI) in the same attack chain
