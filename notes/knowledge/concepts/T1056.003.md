# T1056.003

## What it is
Like a corrupt postal worker who secretly photocopies every letter passing through their hands, Web Portal Capture intercepts credentials as users submit them through legitimate-looking login pages. Precisely defined, it is the technique where adversaries modify web portals or deploy fake login pages to capture credentials submitted by users, often by tampering with server-side code on a compromised web application.

## Why it matters
During the 2020 SolarWinds campaign, attackers who gained access to internal infrastructure were positioned to modify web-facing portals to silently harvest credentials as legitimate administrators logged in — turning trusted tools into traps. Defenders counter this by implementing file integrity monitoring (FIM) on web application directories to detect unauthorized modifications to login page source code.

## Key facts
- Attackers typically modify PHP, ASP, or JavaScript files on already-compromised web servers to exfiltrate credentials to an attacker-controlled endpoint
- Unlike phishing, the victim visits a *genuinely legitimate URL* — the malicious capture happens server-side, making it nearly invisible to the end user
- Captured credentials are often exfiltrated via HTTP POST requests to external IPs or written to hidden log files on the server
- Detection relies on monitoring web logs for unexpected outbound connections from web servers and hashing critical application files
- This is a sub-technique of Input Capture (T1056), alongside keylogging, GUI input capture, and credential API hooking
