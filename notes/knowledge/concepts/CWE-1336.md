# CWE-1336

## What it is
Like handing a stranger a magic wand and saying "only use it for good," template engines that allow unrestricted expression evaluation give attackers a direct line into server-side execution. CWE-1336 is **Improper Neutralization of Special Elements Used in a Template Engine**, where user-controlled input is embedded into a server-side template without sanitization, enabling Server-Side Template Injection (SSTI). The engine obediently executes the attacker's expressions as legitimate code.

## Why it matters
In 2019, researchers found SSTI vulnerabilities in Jinja2-powered Flask applications where user input passed directly into `render_template_string()` — an attacker submitting `{{7*7}}` getting back `49` confirmed execution, and escalating to `{{config.__class__.__init__.__globals__['os'].popen('id').read()}}` achieved full remote code execution. This is a critical-severity path that can compromise an entire server with no authentication required.

## Key facts
- **Template engines affected** include Jinja2 (Python), Twig (PHP), Freemarker (Java), Pebble, and Smarty — each has unique payload syntax
- **Detection fingerprint**: injecting `{{7*7}}`, `${7*7}`, or `<%= 7*7 %>` and observing `49` in the response confirms SSTI
- **Impact ceiling is RCE**: SSTI can traverse Python/Java object hierarchies to reach OS-level commands, making it as dangerous as SQL injection
- **Root cause**: developers using dynamic template rendering (e.g., `render_template_string(user_input)`) instead of passing data as template variables
- **Mitigation**: use sandboxed template environments, pass user data as context variables (never concatenate into template strings), and apply strict input allowlisting
