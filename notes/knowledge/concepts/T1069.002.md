# T1069.002

## What it is
Like a new employee sneaking a peek at the org chart to find out who holds the keys to the executive washroom, an attacker queries Active Directory to discover domain groups and their members. Precisely, T1069.002 is the enumeration of **domain-level permission groups** (e.g., Domain Admins, Enterprise Admins) to identify high-value accounts and privilege structures within a Windows domain environment.

## Why it matters
During the 2020 SolarWinds breach, operators used LDAP queries and tools like `AdFind` to enumerate domain group memberships after initial access — mapping exactly which accounts could move laterally or access crown-jewel systems. Defenders who monitor for anomalous LDAP enumeration queries (Event ID 4661, 4798, 4799) can catch this reconnaissance before privilege escalation occurs.

## Key facts
- **Common tools used**: `net group /domain`, `AdFind.exe`, `BloodHound/SharpHound`, and PowerShell's `Get-ADGroupMember`
- **Key Windows Event IDs to monitor**: 4661 (object handle requested), 4798 (user's local group membership enumerated), 4799 (security-enabled local group membership enumerated)
- **BloodHound** specifically ingests domain group data to graph attack paths to Domain Admin — making it the go-to tool for both red and blue teams
- **Stealth technique**: Attackers often use LDAP queries over port 389/636 because they blend with normal AD traffic, making them harder to filter than `net` commands
- **MITRE tactic**: Discovery (TA0007) — this is reconnaissance *inside* the network, post-initial access, not external scanning
