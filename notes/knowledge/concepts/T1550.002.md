# T1550.002

## What it is
Like using a wax impression of a hotel key to open a room without ever stealing the original, Pass the Hash lets an attacker authenticate as a user by using their stolen password hash — never needing to crack it into plaintext. It exploits NTLM authentication, where the hash *is* the credential: Windows will accept it directly to prove identity.

## Why it matters
In the 2014 Sony Pictures breach, attackers moved laterally across the network by harvesting NTLM hashes from memory and replaying them against other systems — compromising machines at scale without a single password being cracked. Defenders who spot unusual lateral movement from service accounts during off-hours are often catching exactly this technique in action.

## Key facts
- Targets **NTLM authentication** specifically; Kerberos requires the plaintext password (or a ticket), making it less directly vulnerable to this exact technique
- Hashes are extracted from **LSASS process memory** using tools like **Mimikatz** (`sekurlsa::pth`) or **Metasploit's `psexec` module**
- **Local Administrator accounts sharing the same password hash** across machines is the primary enabler — one hash unlocks many doors
- Mitigated by enabling **Protected Users Security Group**, disabling NTLM where possible, and deploying **Credential Guard** to virtualize LSASS
- **KB2871997** (Windows patch) reduces but does not eliminate exposure; it prevents caching of certain credentials but NTLM hashes can still be captured in some configurations
