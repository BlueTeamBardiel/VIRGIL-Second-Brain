# CWE-377

## What it is
Like writing sensitive notes on a napkin in a busy café and leaving it on the table while you step away — anyone can read or swap it before you return. CWE-377 is the insecure creation of temporary files, where an application creates predictable or world-readable temp files in shared directories (like `/tmp`), allowing attackers to access, modify, or replace those files before the application uses them.

## Why it matters
A classic exploitation scenario is the **symlink attack**: an attacker predicts the temp file name (e.g., `/tmp/app12345.tmp`), creates a symbolic link from that path to a sensitive system file like `/etc/passwd`, and when the vulnerable application writes to its "temp file," it overwrites the system file instead. This was a real attack vector in early Unix daemons and still appears in poorly written scripts and legacy applications.

## Key facts
- The root cause is typically using predictable filenames (PID-based, timestamp-based) combined with writing to world-writable shared directories like `/tmp` or `/var/tmp`
- **Race condition window (TOCTOU)** is the core mechanism: time-of-check to time-of-use gap lets attackers swap the file between creation and use
- Secure fix: use `mkstemp()` in C or equivalent OS-provided functions that atomically create files with random names and restrictive permissions (mode 0600)
- On Linux, the `O_EXCL | O_CREAT` flag combination prevents overwriting existing files, closing the race window
- CWE-377 is a child of CWE-378 (Creation of Temporary File With Insecure Permissions) and relates closely to CWE-362 (Race Condition)
