# SMM

## What it is
Think of SMM as a secret basement in a house — the occupants (OS, hypervisors, even antivirus) have no idea it exists, yet it controls the building's power and locks. System Management Mode is a highly privileged x86 CPU operating mode (Ring -2), activated via a System Management Interrupt (SMI), that runs firmware code in isolated memory (SMRAM) completely invisible to the operating system.

## Why it matters
In 2015, researchers demonstrated "ThinkPwn" and similar SMM rootkits that exploited vulnerable SMM handlers in Lenovo firmware — once code executes in SMM, it persists across OS reinstalls, bypasses Secure Boot, and can disable write protections on flash memory to permanently backdoor a system. Defenders counter this by enabling SMM Lock bits and enforcing UEFI Secure Boot with verified firmware updates, but patching SMM vulnerabilities requires firmware updates that many users never apply.

## Key facts
- SMM operates at **Ring -2**, below the hypervisor (Ring -1) and OS kernel (Ring 0), making it the most privileged execution environment on x86 systems
- SMM code and data reside in **SMRAM**, a protected memory region that the OS cannot read or write while SMM is active
- Triggered exclusively by a **System Management Interrupt (SMI)**, which can be generated by hardware events or software writes to specific I/O ports
- **SMRAM Lock** (D_LOCK bit in chipset registers) prevents remapping of SMRAM after boot — failure to set this bit is a classic SMM vulnerability class
- SMM rootkits are particularly dangerous because they **survive full OS reinstallation** and can undermine hypervisor-based security controls like Credential Guard
