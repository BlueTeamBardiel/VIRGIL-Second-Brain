# RFC 3164

## What it is
Think of it as a postal system where anyone can drop a letter in the mailbox with no return address verification — the recipient just trusts the sender wrote their name correctly. RFC 3164 is the original BSD syslog protocol standard, defining how network devices and systems transmit log messages over UDP port 514 to a centralized syslog server, with no authentication or encryption.

## Why it matters
An attacker who gains a foothold on a network segment can perform **syslog spoofing** — injecting forged log entries to a SIEM or syslog collector that falsely exonerate malicious activity or frame a legitimate host. Because RFC 3164 has no message integrity verification, a defender cannot distinguish a real log from a crafted one without additional transport-layer controls like TLS (addressed in RFC 5425).

## Key facts
- Uses **UDP port 514** — connectionless, meaning no delivery confirmation and no session to authenticate
- Messages include a **PRI value** (priority = Facility × 8 + Severity), with 8 facility levels and severity ranging 0 (Emergency) to 7 (Debug)
- Maximum message length is **1024 bytes**; anything longer is silently truncated
- **No encryption, no authentication, no integrity checking** — all three CIA triad controls for the log data itself are absent
- Superseded by **RFC 5424** (structured syslog) and secured transport defined in **RFC 5425** (syslog over TLS); RFC 3164 is formally classified as **"Historic"** by the IETF
