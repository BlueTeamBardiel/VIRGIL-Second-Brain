# Certified Cloud Security Engineer

## What it is
Like a structural engineer who doesn't just design buildings but specifically ensures skyscrapers can withstand earthquakes, wind loads, and fire — a Certified Cloud Security Engineer (C|CSE) is a practitioner trained not just in security, but in securing dynamic, multi-tenant cloud infrastructure across AWS, Azure, and GCP. It is a vendor-neutral, EC-Council certification validating skills in cloud security architecture, governance, incident response, and penetration testing within cloud environments.

## Why it matters
In the 2019 Capital One breach, a misconfigured AWS WAF allowed a threat actor to exploit Server-Side Request Forgery (SSRF) to steal IAM role credentials from the EC2 metadata service, exposing over 100 million customer records. A C|CSE-trained engineer would recognize the principle of least privilege violations, metadata service hardening requirements (IMDSv2 enforcement), and detective controls like AWS CloudTrail that could have flagged the anomalous credential usage before exfiltration completed.

## Key facts
- C|CSE covers 5 major cloud platforms: AWS, Azure, GCP, Oracle, and private cloud environments — making it broader than single-vendor certifications like AWS Security Specialty
- Exam domains include cloud security design, data security, identity and access management (IAM), incident response, and cloud application security
- The certification addresses shared responsibility model implementation — a concept directly testable on Security+ and CySA+
- C|CSE includes hands-on labs in security automation using tools like Terraform, CloudFormation, and SIEM integration with cloud-native logging
- Issued by EC-Council; requires either 2 years of information security experience or completion of the official training before attempting the exam
