# T1055

## What it is
Like a puppeteer slipping their hand inside a marionette, process injection is when an attacker inserts malicious code into a legitimate, already-running process. This allows the malicious code to execute under the identity and permissions of that trusted process, bypassing security controls that monitor process creation.

## Why it matters
In the 2020 SolarWinds supply chain attack, adversaries used process injection techniques to execute malicious payloads inside trusted Windows processes like `svchost.exe`, making their activity blend into normal system telemetry. Security analysts hunting for IOCs had to look for memory anomalies rather than suspicious process names, illustrating why behavioral detection is critical over signature-based tools alone.

## Key facts
- **Sub-techniques include**: DLL Injection (T1055.001), Process Hollowing (T1055.012), Thread Execution Hijacking (T1055.003), and PTR Injection — each uses a different mechanism to load code into target memory.
- **Process Hollowing** unmaps a legitimate process's code and replaces it with malicious code, making it especially stealthy since the process name remains trusted.
- **Detection focus**: Analysts look for unusual API call sequences — `OpenProcess` → `VirtualAllocEx` → `WriteProcessMemory` → `CreateRemoteThread` — this chain is a classic injection signature.
- **Defense evasion AND privilege escalation**: T1055 serves dual MITRE ATT&CK purposes — it hides malware AND can inherit elevated privileges from the target process.
- **EDR tools** detect injection by monitoring memory permissions changes (e.g., regions marked `RWX` — readable, writable, and executable simultaneously) which is abnormal for legitimate processes.
