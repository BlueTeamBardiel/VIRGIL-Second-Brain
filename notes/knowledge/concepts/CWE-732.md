# CWE-732

## What it is
Like leaving your house's front door unlocked because you assumed only family would ever walk by — Incorrect Permission Assignment for Critical Resource occurs when a file, directory, or resource is granted overly permissive access rights, allowing unintended actors to read, modify, or execute it. The flaw is in the *assignment*, not the absence, of permissions.

## Why it matters
In 2022, misconfigured Docker socket permissions (`/var/run/docker.sock` set world-readable/writable) allowed low-privileged container users to issue Docker commands to the host, enabling full container escape and root-level host compromise. Proper assignment — restricting socket access to the `docker` group only — would have neutralized the attack vector entirely.

## Key facts
- CWE-732 is distinct from CWE-276 (Incorrect Default Permissions) — 732 covers *any* incorrect assignment, not just defaults set during installation
- Common vulnerable targets: cron job scripts, SSH `authorized_keys` files, configuration files with embedded credentials, and Unix domain sockets
- On Linux, the dangerous pattern is `chmod 777` on sensitive files — world-writable means any local user can overwrite or inject content
- CVSS scores for 732 instances frequently range 7.0–9.8 when the misconfigured resource provides a direct privilege escalation path
- Detection tools include `find / -perm -o+w -type f` (world-writable files) and automated scanners like Lynis or OpenSCAP that audit filesystem permissions against benchmarks
