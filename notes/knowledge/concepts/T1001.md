# T1001

## What it is
Like a spy hiding a message inside a mundane postcard about the weather, data obfuscation disguises command-and-control (C2) traffic so it blends into normal network activity. Precisely, T1001 describes adversary techniques that encode, encrypt, or otherwise manipulate C2 communications to evade detection by network monitoring tools.

## Why it matters
During the APT41 campaign, attackers encoded C2 traffic using custom XOR routines embedded inside what appeared to be standard HTTP responses, allowing malware to receive instructions for months before analysts noticed the anomalous byte patterns. A defender using deep packet inspection or behavioral baselining could flag the statistical irregularities in packet payloads even without decrypting the content.

## Key facts
- T1001 is the parent technique with three sub-techniques: **T1001.001** (Junk Data), **T1001.002** (Steganography), and **T1001.003** (Protocol Impersonation)
- Junk data involves padding C2 traffic with random bytes to defeat signature-based detection and throw off anomaly baselines
- Steganography sub-technique hides commands inside images, audio, or documents transmitted over legitimate channels — a classic SUNBURST-era tactic
- Protocol Impersonation makes malicious traffic mimic legitimate protocols (e.g., fake DNS or HTTPS) without actually using them, often detectable by certificate anomalies or malformed headers
- Detection strategy: focus on **entropy analysis** of payloads, **beacon interval jitter**, and **unusual user-agent strings** rather than relying solely on known signatures
