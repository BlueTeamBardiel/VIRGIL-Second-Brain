# CWE-319

## What it is
Like shouting your bank PIN across a crowded restaurant instead of whispering it — CWE-319 occurs when sensitive data is transmitted over a network channel that lacks encryption, exposing it to anyone who can observe the traffic. This weakness specifically covers cleartext transmission of sensitive information, where an attacker with network access can read credentials, session tokens, or PII without any cryptographic effort.

## Why it matters
In 2021, security researchers discovered many IoT devices (including home routers and cameras) sending credentials over plain HTTP or Telnet during setup. An attacker on the same Wi-Fi segment could run Wireshark, capture the admin password in a single packet, and gain full device control — no cracking required. Fixing it is straightforward: enforce TLS/HTTPS and disable unencrypted protocol fallback.

## Key facts
- **Common culprits**: HTTP instead of HTTPS, FTP instead of SFTP/FTPS, Telnet instead of SSH, LDAP instead of LDAPS, SMTP without STARTTLS
- **Attack method**: Passive network sniffing (e.g., Wireshark, tcpdump) on a shared or compromised network segment — no active exploitation needed
- **CVSS impact**: Classified under Confidentiality impact; severity scales with sensitivity of data exposed (credentials = Critical, generic metadata = Low)
- **Detection**: Security scanners like Nessus flag services listening on unencrypted ports (21, 23, 80, 389); TLS configuration checkers like testssl.sh identify missing or fallback encryption
- **Compliance relevance**: Violates PCI-DSS Requirement 4.2 (never send PANs over unencrypted channels) and HIPAA Technical Safeguards requiring ePHI transmission security
