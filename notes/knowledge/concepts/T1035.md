# T1035

## What it is
Like a burglar who breaks in once but then uses a spare key hidden under the mat for every visit after — Service Execution is when attackers abuse the Windows Service Control Manager (SCM) to run malicious code as a service, giving them a persistent, privileged execution channel that blends in with legitimate system processes.

## Why it matters
During the NotPetya campaign, attackers used PsExec (which itself leverages service execution) to laterally move across networks by remotely creating and starting services on victim machines. Defenders monitoring for unexpected `sc.exe` calls or new service registrations in the registry could have detected this movement before widespread encryption occurred.

## Key facts
- T1035 maps to **Service Execution** in the MITRE ATT&CK framework under the **Execution** tactic
- Attackers commonly use `sc.exe`, `PsExec`, or PowerShell's `New-Service` cmdlet to create or interact with services
- Services run under `SYSTEM` by default if not configured otherwise, giving attackers immediate high-privilege execution
- Malicious services are often registered at `HKLM\SYSTEM\CurrentControlSet\Services\` — a key hunting ground for forensic analysts
- This technique overlaps with **Persistence** when a service is set to auto-start on boot (`START_TYPE = AUTO_START`)
