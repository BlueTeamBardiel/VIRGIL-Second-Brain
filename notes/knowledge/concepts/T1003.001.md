# T1003.001

## What it is
Think of LSASS as a bank vault that keeps everyone's PIN numbers in memory — and this technique is someone photographing the inside of that vault. LSASS Memory Dumping is the act of extracting credentials from the Local Security Authority Subsystem Service (lsass.exe) process memory on Windows systems. Attackers dump this process to harvest plaintext passwords, NTLM hashes, and Kerberos tickets stored there during active sessions.

## Why it matters
In the 2020 SolarWinds intrusion, threat actors used tools like Mimikatz against LSASS to harvest credentials and move laterally across victim networks with legitimate-looking authentication. Defenders countering this configure Windows Credential Guard, which isolates LSASS in a virtualization-based security enclave, making the memory contents unreadable even if dumped.

## Key facts
- **Mimikatz** (`sekurlsa::logonpasswords`) is the canonical tool for this technique; ProcDump (`procdump.exe -ma lsass.exe`) is a common LOLBin alternative
- Windows **Protected Process Light (PPL)** can be enabled for LSASS to block most user-mode dumping attempts
- **Credential Guard** (Windows 10+) moves credential secrets into a hypervisor-protected region, rendering LSASS dumps credential-free
- Detection signals include: processes opening `PROCESS_VM_READ` handle to lsass.exe, unexpected minidump files (`.dmp`), and Event ID **10** in Sysmon logs
- NTLM hashes captured this way can be used in **Pass-the-Hash** attacks without ever cracking the actual password
