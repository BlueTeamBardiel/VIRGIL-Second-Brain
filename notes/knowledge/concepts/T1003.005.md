# T1003.005

## What it is
Like a locksmith making a wax impression of a master key without ever touching the original, this technique copies cached domain credentials from a password manager built into Windows itself. T1003.005 refers to **Cached Domain Credentials** dumping — extracting the MSCache (Domain Cached Credentials) hashes stored locally so Windows can authenticate users when a domain controller is unreachable.

## Why it matters
During the 2020 SolarWinds campaign, attackers moved laterally through environments by dumping cached credentials on workstations that had been used by privileged users — even when those machines were offline from the domain. An attacker with local admin rights can extract these hashes using tools like `secretsdump.py` and then crack them offline with hashcat, bypassing any need to touch Active Directory directly.

## Key facts
- Windows stores the last 10 domain logons by default (configurable via `CachedLogonsCount` registry key at `HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon`)
- Cached hashes use the **MS-Cache v2 (DCC2)** format — they are *not* NTLM hashes and cannot be used in Pass-the-Hash attacks; they must be cracked
- DCC2 hashes are salted with the lowercase username, making them resistant but not immune to offline brute-force
- Extraction requires **SYSTEM-level privileges**; common tools include Mimikatz (`lsadump::cache`) and Impacket's `secretsdump`
- Defenders can set `CachedLogonsCount` to **0** on servers and privileged workstations to eliminate this attack surface entirely
