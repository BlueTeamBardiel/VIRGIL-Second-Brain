# T1021.001

## What it is
Like a master key that opens the front door of every Windows machine on the network, Remote Desktop Protocol (RDP) lets attackers sit down at a remote system's graphical desktop as if they were physically present. Precisely, T1021.001 describes adversaries leveraging RDP (TCP port 3389) to move laterally or establish persistent access by authenticating to remote systems using valid credentials or stolen hashes.

## Why it matters
During the 2021 Colonial Pipeline ransomware attack, the initial intrusion vector was a compromised VPN account — but RDP exposure is the canonical example of how ransomware operators like REvil and Conti pivot laterally once inside a network. An attacker with a single set of domain credentials can RDP hop across dozens of servers, deploying ransomware payloads or exfiltrating data while blending in with legitimate administrator activity.

## Key facts
- RDP operates on **TCP port 3389** by default; defenders often monitor this port for unauthorized inbound connections
- Attackers frequently use **Pass-the-Hash** attacks against RDP when Network Level Authentication (NLA) is disabled, bypassing the need for plaintext passwords
- **Credential stuffing** against internet-exposed RDP is so common that Shodan regularly indexes millions of publicly accessible RDP endpoints
- Windows Event ID **4624** (logon type 10 = RemoteInteractive) is the primary log artifact for detecting RDP authentication events
- Enabling **Restricted Admin Mode** prevents credential exposure on the remote host but can be abused for Pass-the-Hash lateral movement itself
