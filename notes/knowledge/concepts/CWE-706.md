# CWE-706

## What it is
Like a bouncer who checks IDs at the wrong door — letting people into the VIP room based on where they *say* they came from rather than verifying it themselves — CWE-706 describes using an incorrectly-resolved name or reference. Specifically, a system resolves a name (file path, URL, hostname) to a resource but uses a different, attacker-controlled resource than the one that was actually validated.

## Why it matters
A classic example is a TOCTOU (Time-of-Check to Time-of-Use) symlink attack: a program checks that `/tmp/safefile.txt` is safe to write, but between the check and the write, an attacker replaces it with a symlink pointing to `/etc/shadow`. The name resolved correctly at check time, but at use time it pointed somewhere else entirely — the validation and the action operated on different resources.

## Key facts
- CWE-706 is categorized under **improper resolution of names or references**, a parent of vulnerabilities like path traversal and symlink attacks
- The core flaw is a **race condition window** or **redirection** between validation and use — "TOCTOU" (Time-of-Check Time-of-Use) is the canonical mechanism
- Attackers exploit this by **swapping the resolved resource** (symlink substitution, DNS rebinding, redirect chains) after validation passes
- **DNS rebinding** is a network-level instance: a hostname resolves to a safe IP during the security check, then resolves to an internal IP when the actual request is made
- Mitigations include **atomic operations** (open-then-stat instead of stat-then-open), using **file descriptors instead of paths** after validation, and pinning DNS responses with short TTL controls
