# T1036.005

## What it is
Like a criminal wearing a police uniform to walk past checkpoints unchallenged, this technique involves naming malware after legitimate system executables. T1036.005 (Match Legitimate Name or Location) is the practice of disguising malicious files by giving them the same name as trusted binaries — like `svchost.exe` or `explorer.exe` — and placing them in expected or nearby directories to evade detection.

## Why it matters
During the NotPetya attack, malicious components were dropped with names mimicking Windows system processes to avoid raising flags in process lists viewed by administrators. Defenders must go beyond name-checking and validate file hashes, digital signatures, and execution paths — because a `svchost.exe` running from `C:\Users\Temp\` is a major red flag, regardless of its name.

## Key facts
- Legitimate Windows binaries like `svchost.exe` live in `C:\Windows\System32\`; the same filename in any other path is immediately suspicious.
- Attackers often place look-alike binaries in `C:\Windows\`, `C:\Temp\`, or user profile directories to exploit lax path-based allow-lists.
- Detection relies on **parent-child process relationship analysis** — `svchost.exe` should only be spawned by `services.exe`.
- Digital signature verification is a primary defense; legitimate Microsoft binaries are signed, while malicious clones typically are not.
- This sub-technique falls under the broader **Masquerading (T1036)** tactic, which is classified under **Defense Evasion** in the MITRE ATT&CK framework.
