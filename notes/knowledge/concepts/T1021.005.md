# T1021.005

## What it is
Like a maintenance worker using a building's hidden service elevator instead of the main lobby, attackers use VNC (Virtual Network Computing) to access a remote desktop through a back channel that many defenders overlook. VNC is a graphical desktop-sharing protocol that allows full visual control of a remote machine, transmitting keyboard and mouse input over the network. Unlike RDP, VNC is platform-agnostic and operates independently of Windows authentication infrastructure.

## Why it matters
During the 2021 ransomware surge, threat actors including groups deploying Ryuk routinely scanned for exposed VNC servers (default port 5900) with weak or no passwords to establish persistent graphical access deep inside victim networks. Because VNC traffic can blend in with legitimate remote administration, defenders relying solely on RDP monitoring would miss the lateral movement entirely — making VNC an ideal pivot point after initial access.

## Key facts
- VNC operates by default on **TCP port 5900** (display :0); each additional display increments the port (5901, 5902, etc.)
- Many VNC implementations store passwords in **weak, reversible encryption** (DES-based 8-character limit), making credential theft straightforward
- Attackers frequently install VNC as a **persistence mechanism** post-exploitation — tools like TightVNC and UltraVNC are commonly abused
- VNC itself provides **no native encryption** unless tunneled through SSH or a VPN, making credentials and session data visible in plaintext on the wire
- Detection focus: look for **unexpected processes** binding to port 5900 and outbound connections to that port from non-admin workstations
