# T1218.012

## What it is
Like a locksmith using the building's own master key to bypass security rather than picking the lock, `verclsid.exe` is a legitimate Windows binary that attackers weaponize to execute malicious COM objects without triggering standard defenses. It is a signed Microsoft executable designed to validate COM class objects, making it a trusted "living off the land" binary (LOLBin) that proxy-executes attacker-controlled code under the cover of Windows legitimacy.

## Why it matters
In a red team engagement, an attacker can invoke `verclsid.exe /S /C {malicious-CLSID}` to instantiate a COM object that downloads and executes a payload — bypassing application whitelisting solutions like AppLocker that trust Microsoft-signed binaries by default. Defenders monitoring only for unsigned executables or unknown processes would miss this entirely, since `verclsid.exe` appears completely normal in process telemetry.

## Key facts
- **Binary path:** `C:\Windows\System32\verclsid.exe` — Microsoft-signed, present on all modern Windows systems
- **Attack syntax:** `verclsid.exe /S /C {CLSID}` where the CLSID points to a malicious registered COM object
- **Defense evasion mechanism:** Inherits trust from its Microsoft signature, bypassing application control policies that whitelist by publisher
- **Detection pivot:** Monitor for `verclsid.exe` spawning unexpected child processes or making unusual network connections — normal behavior involves no child processes
- **MITRE parent technique:** T1218 — System Binary Proxy Execution; this is one of ~14 sub-techniques covering LOLBins like `msiexec`, `regsvr32`, and `rundll32`
