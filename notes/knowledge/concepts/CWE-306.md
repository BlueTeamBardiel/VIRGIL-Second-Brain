# CWE-306

## What it is
Like a bank vault that opens just because you *know where it is* — no key, no PIN, no ID required — CWE-306 describes a system that performs sensitive operations without ever verifying who is asking. Formally: Missing Authentication for Critical Function occurs when software does not authenticate users before granting access to functionality that should be restricted.

## Why it matters
In 2021, attackers exploited unauthenticated API endpoints in Verkada's security camera platform, accessing live feeds from 150,000 cameras including in hospitals and prisons — no credentials needed, just a discovered admin endpoint. Proper authentication gates on every privileged function, not just the login page, would have blocked this entirely.

## Key facts
- CWE-306 is distinct from *broken* authentication (CWE-287) — here, authentication is simply **absent**, not just weak or bypassable
- Common in IoT devices, industrial control systems (ICS/SCADA), and internal microservices that assume network location implies trust
- Often surfaces in APIs where developers secure the UI but forget the underlying endpoints are directly reachable
- CVSS scores for CWE-306 vulnerabilities frequently reach **9.8 (Critical)** because no user interaction or privileges are required to exploit
- Listed in both the OWASP API Security Top 10 (API2: Broken Authentication) and CWE Top 25 Most Dangerous Software Weaknesses
