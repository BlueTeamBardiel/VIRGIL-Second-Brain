# T1071

## What it is
Like a smuggler hiding contraband inside a delivery truck marked "USPS," attackers hide malicious command-and-control traffic inside protocols that firewalls routinely wave through — HTTP, HTTPS, DNS, or SMTP. T1071 (Application Layer Protocol) is the MITRE ATT&CK technique describing adversaries using standard, legitimate application-layer protocols to blend C2 communications into normal network traffic.

## Why it matters
During the SolarWinds SUNBURST attack, malware communicated with attacker infrastructure by crafting DNS queries that mimicked legitimate Orion telemetry traffic — security teams saw "normal" DNS lookups while exfiltrated data and commands flowed freely. Defenders countering T1071 focus on anomaly detection: flagging unusual DNS query volumes, unexpected HTTPS destinations, or beaconing patterns (regular, clock-like connection intervals) rather than blocking protocols outright.

## Key facts
- **Sub-techniques matter for exams:** T1071.001 (Web Protocols/HTTP/S), T1071.002 (File Transfer Protocols), T1071.003 (Mail Protocols), T1071.004 (DNS)
- **Beaconing** is the telltale behavioral signature — malware "checks in" at suspiciously regular intervals, detectable via netflow analysis
- **DNS tunneling** (T1071.004) encodes C2 data inside DNS query strings; tools like `dnscat2` and `iodine` automate this
- **HTTPS C2** is particularly stealthy because payload encryption defeats content inspection — defenders rely on certificate anomalies (self-signed certs, unusual SANs) and JA3 fingerprinting
- Detection leverages **network traffic analysis (NTA)** tools and SIEM correlation rules looking for long-duration sessions, high query frequency, or large DNS response sizes
