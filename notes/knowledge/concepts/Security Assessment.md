# Security Assessment

## What it is
Like a doctor ordering a full-body scan before surgery — not to fix anything yet, just to understand what's there — a security assessment is a structured evaluation of an organization's security posture to identify vulnerabilities, misconfigurations, and control gaps. It encompasses multiple techniques (scanning, interviewing, reviewing policies) and produces a prioritized picture of risk before harm occurs.

## Why it matters
In 2020, the SolarWinds compromise succeeded partly because organizations lacked visibility into their own supply chain dependencies — a gap a thorough security assessment would have surfaced. Companies that had conducted recent third-party assessments were faster to identify affected systems and isolate compromised Orion instances, cutting dwell time significantly.

## Key facts
- **Assessment ≠ Penetration Test**: Assessments identify and evaluate risks; pen tests actively exploit them. CompTIA distinguishes these sharply — know the difference.
- **Types include**: vulnerability assessment, risk assessment, threat assessment, security audit, and posture assessment — each with distinct scope and methodology.
- **Credentialed vs. non-credentialed scans**: Credentialed scans authenticate to systems and find more vulnerabilities; non-credentialed simulate an external attacker's view.
- **Assessment phases**: Planning → Data Collection → Analysis → Reporting → Remediation Verification — the loop closes with retesting.
- **Compliance drivers**: PCI-DSS requires quarterly vulnerability scans by an Approved Scanning Vendor (ASV); HIPAA mandates periodic risk assessments as a required implementation specification.
