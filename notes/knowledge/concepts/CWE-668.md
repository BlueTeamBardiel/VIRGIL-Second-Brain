# CWE-668

## What it is
Like leaving your diary on a park bench "just temporarily" while you grab coffee — the resource is accessible to far more people than intended. CWE-668 is the exposure of a resource to the wrong sphere of control, meaning software makes a resource available to actors outside its intended access boundary. This includes files, memory, network ports, or credentials that end up reachable by untrusted parties.

## Why it matters
In 2021, a misconfigured AWS S3 bucket (a textbook CWE-668 instance) exposed 3TB of Twitch's source code and internal credentials when the storage resource was inadvertently made publicly accessible. An attacker enumerating open buckets could harvest API keys and pivot deeper into Twitch's infrastructure — no exploitation required, just walking through an unlocked door.

## Key facts
- **Scope is the core issue**: the resource itself may be legitimate; the problem is *who* can reach it — wrong network zone, wrong user, wrong process
- Commonly manifests as world-readable files (`chmod 777`), open network ports bound to `0.0.0.0` instead of `127.0.0.1`, or temporary files in shared `/tmp` directories
- Distinguished from CWE-200 (Information Exposure) in that CWE-668 focuses on *control sphere violations*, not just information leakage
- Frequently precedes privilege escalation attacks — an attacker reads a resource they shouldn't, finds credentials, then escalates
- Detection methods include attack surface analysis, file permission audits, and network port scanning from an untrusted perspective
