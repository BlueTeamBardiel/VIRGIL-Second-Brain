# T1222

## What it is
Like a burglar changing the locks on your house so the owner can't get back in, file and directory permission modification lets attackers rewrite who's allowed to access what. Precisely, T1222 describes adversaries modifying access control permissions on files, folders, or registries to enable further malicious activity or block defenders from recovering systems.

## Why it matters
During the 2021 ransomware wave, attackers routinely used `icacls` on Windows to strip administrators of file access rights before deploying encryption payloads — ensuring that even if defenders detected the intrusion, they couldn't cleanly remove malicious files or restore backups without fighting through permission walls. Defenders watching for unexpected `chmod 777` on Linux or `icacls /grant Everyone:F` on Windows can catch this behavior early in the kill chain.

## Key facts
- **Windows tools abused**: `icacls`, `cacls`, `attrib`, and PowerShell's `Set-Acl` cmdlet are the primary native utilities attackers leverage
- **Linux tools abused**: `chmod`, `chown`, and `chattr` (especially `chattr +i` to make files immutable and prevent deletion)
- **Sub-techniques**: T1222.001 covers Windows File/Directory Permissions; T1222.002 covers Linux/Mac equivalents — the split matters for exam precision
- **Detection signature**: Bulk or recursive permission changes on sensitive directories (e.g., `/etc/`, `C:\Windows\System32\`) by non-admin processes are high-fidelity alerts
- **MITRE tactic**: Defense Evasion — the goal is usually to hide malware, disable security tools, or prevent cleanup rather than gain initial access
