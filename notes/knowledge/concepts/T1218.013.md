# T1218.013

## What it is
Like hiring a locksmith to break into your own house so no one suspects you of breaking in, attackers use **mavinject.exe** — a legitimate Windows tool — to inject malicious code into running processes. Mavinject is a signed Microsoft binary designed to inject DLLs into processes, but adversaries abuse it to execute arbitrary code under the cover of a trusted executable.

## Why it matters
In documented APT campaigns, attackers have used mavinject.exe to inject malicious DLLs into suspended processes like `waitfor.exe`, effectively bypassing application whitelisting controls that trust Microsoft-signed binaries. Defenders watching for unsigned code execution would miss this entirely, making it a potent evasion technique against endpoint controls.

## Key facts
- **Mavinject.exe** lives at `C:\Windows\System32\mavinject.exe` and is a legitimate Microsoft-signed binary, making it a classic **Living off the Land Binary (LOLBin)**
- The core syntax abused by attackers: `mavinject.exe <PID> /INJECTRUNNING <path_to_dll>` — injecting a DLL into an already-running process by PID
- It falls under **T1218 - System Binary Proxy Execution**, specifically the sub-technique for signed binary abuse
- Detection focus: alert on `mavinject.exe` spawned by unusual parent processes (e.g., Office apps, scripting engines) or injecting into non-standard target processes
- Mitigation: restrict execution via **Software Restriction Policies** or **AppLocker** rules targeting mavinject.exe specifically; most environments have zero legitimate need for it
