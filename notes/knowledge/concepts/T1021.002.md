# T1021.002

## What it is
Think of SMB/Windows Admin Shares like a master key that building maintenance staff use to access every apartment — useful for legitimate work, devastating if stolen. Precisely, this technique involves adversaries using valid credentials to authenticate over SMB (port 445) and connect to hidden administrative shares like `C$`, `ADMIN$`, or `IPC$` that Windows automatically creates on every system.

## Why it matters
During the NotPetya outbreak, the malware used stolen credentials combined with EternalBlue/Mimikatz to authenticate to `ADMIN$` shares across the network, dropping and executing its payload on thousands of machines within minutes. Defenders who had disabled unnecessary admin shares or blocked lateral SMB traffic at internal firewalls significantly limited propagation.

## Key facts
- Default Windows admin shares: `C$` (root of C drive), `ADMIN$` (Windows directory), `IPC$` (inter-process communication) — all hidden with the `$` suffix
- Requires valid credentials with local administrator privileges on the target machine
- Common tools used for execution over these shares: `PsExec`, `wmic`, `net use`, and Impacket's `smbexec`
- Detection signature: look for Event ID 5140 (network share access) and 5145 (share object access check) in Windows Security logs
- Disabling admin shares via registry (`AutoShareWks = 0`) breaks legitimate management tools like SCCM, making it a double-edged defense
