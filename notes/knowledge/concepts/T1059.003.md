# T1059.003

## What it is
Like a janitor who has master keys to every room and never gets questioned, Windows Command Shell (cmd.exe) is a trusted, pre-installed interpreter that attackers abuse because it's already there and rarely blocked. Precisely defined, T1059.003 describes adversary use of cmd.exe to execute commands, scripts, and batch files to perform malicious actions on a Windows system.

## Why it matters
During the 2020 SolarWinds campaign, attackers used cmd.exe to execute reconnaissance commands and invoke PowerShell payloads after gaining initial access via the SUNBURST backdoor — blending into normal administrative traffic. Defenders who monitor for unusual cmd.exe parent processes (like a web server spawning cmd.exe) can detect this technique before lateral movement begins.

## Key facts
- Cmd.exe can execute inline commands (`/c` flag), batch files (`.bat`, `.cmd`), and chain commands using `&`, `&&`, or `|` operators
- A classic red flag: cmd.exe spawned by unusual parent processes such as `outlook.exe`, `iexplore.exe`, or `w3wp.exe` (IIS worker)
- Attackers use cmd.exe to disable defenses: `netsh advfirewall set allprofiles state off` or `sc stop WinDefend`
- Logging cmd.exe activity requires **Windows Security Audit Policy** for process creation (Event ID 4688) with command-line auditing enabled — off by default
- T1059.003 often chains into T1059.001 (PowerShell) since cmd.exe is used to invoke `powershell.exe -EncodedCommand`
