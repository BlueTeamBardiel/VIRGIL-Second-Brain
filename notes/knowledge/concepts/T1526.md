# T1526

## What it is
Like a burglar who, before breaking in, walks the neighborhood to find which houses have security cameras, alarm company signs, and what kind of locks are installed — an attacker queries cloud provider APIs to map what services and infrastructure are available. T1526 (Cloud Service Discovery) is the technique where adversaries enumerate cloud services running in a victim's environment (e.g., AWS, Azure, GCP) to understand the attack surface and identify high-value targets like storage buckets, databases, or serverless functions.

## Why it matters
In the Capital One breach (2019), the attacker exploited an SSRF vulnerability and then used AWS metadata and service enumeration to discover accessible S3 buckets containing sensitive data — the discovery phase directly enabled the exfiltration. Defenders monitor for unusual or high-volume API calls to list services (e.g., `ListBuckets`, `DescribeInstances`) as early indicators of compromise.

## Key facts
- Attackers commonly use native CLI tools (`aws cli`, `az cli`, `gcloud`) or SDKs to perform enumeration — these look like legitimate admin activity
- Key API calls to monitor: AWS `ListBuckets`, `DescribeInstances`, `ListFunctions`; Azure `az resource list`; GCP `gcloud services list`
- This is a **Discovery** tactic (TA0007) in MITRE ATT&CK, often chained after **Valid Accounts (T1078)** or **Steal Application Access Token (T1528)**
- CloudTrail (AWS), Azure Monitor, and GCP Cloud Audit Logs are the primary detection data sources for this technique
- Excessive API calls in short time windows from new or unusual IAM identities are a reliable behavioral indicator
