# T1055.013

## What it is
Like a parasite that hijacks a host organism's nervous system to puppet its movements, Process Doppelgänging replaces a legitimate process's memory image with malicious code before the OS ever writes it to disk. Specifically, it abuses Windows NTFS Transactions (TxF) to load malicious executable content into a transacted file, map it into memory, then roll back the transaction — leaving no file artifact on disk while the malicious code runs inside what appears to be a trusted process.

## Why it matters
In 2017, the SynAck ransomware gang deployed Process Doppelgänging to evade endpoint detection tools that relied on scanning files-at-rest or monitoring standard process creation APIs. Because the malicious payload was never committed to the filesystem and the process appeared legitimate to the Windows loader, many AV solutions failed to flag it until the encryption had already begun.

## Key facts
- Exploits **NTFS Transactional API (TxF)** — specifically `CreateTransaction`, `CreateFileTransacted`, and `RollbackTransaction` — to stage malicious code without a persistent file artifact
- The technique evades **file-based scanning** because the transacted write is rolled back before the file is flushed to disk
- Differs from Process Hollowing (T1055.012): Doppelgänging uses the Windows loader with a transacted file; Hollowing unmaps a legitimate process's memory post-load
- Detection relies on **behavioral monitoring** of TxF API calls and memory-scanning tools (e.g., Volatility) rather than file hash matching
- Effective against many **userland AV hooks** because code is loaded through the legitimate Windows image loader (`NtCreateProcessEx`)
