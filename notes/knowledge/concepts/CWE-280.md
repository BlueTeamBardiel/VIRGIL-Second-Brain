# CWE-280

## What it is
Like a hotel manager who panics and exposes the master keycard location when a guest asks a confusing question — instead of calmly denying access — this weakness occurs when software mishandles permissions errors in a way that grants *more* access than intended. CWE-280 (Improper Handling of Insufficient Permissions or Privileges) describes when an application fails to correctly respond to a permission denial, accidentally escalating privilege or bypassing access controls.

## Why it matters
A classic manifestation: a Windows service catches an "Access Denied" exception when trying to write to a protected directory, then silently falls back to writing to a world-writable temp directory — which an attacker has pre-positioned a malicious payload in. The error-handling path becomes the attack path, turning a security boundary into a welcome mat.

## Key facts
- **Root cause is in exception/error handling code**, not the primary access-control logic — making it easy to miss in code review
- Commonly paired with **privilege escalation attacks**: the fallback behavior runs under a higher-privilege context than the error warrants
- Differs from CWE-284 (Improper Access Control) — CWE-280 is specifically about *reacting* to a permissions failure incorrectly, not missing access control entirely
- CVSS scores for exploitable instances often reach **7.0–8.5** (High) due to privilege escalation potential
- Mitigated by **fail-secure design**: when permissions are denied, the system should halt the operation entirely — never silently downgrade to a less-secure path
