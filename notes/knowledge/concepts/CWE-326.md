# CWE-326

## What it is
Like building a vault door out of cardboard because it *looks* like steel — the structure exists, but the protection is hollow. CWE-326 is the use of cryptographic algorithms with inadequate key lengths, where encryption is implemented correctly in form but the key size is too short to resist brute-force or mathematical attacks. The cipher runs, data is "encrypted," but the security margin has effectively collapsed.

## Why it matters
In 2012, researchers cracked 512-bit RSA keys used in HTTPS certificates in under 72 hours using cloud computing resources. Organizations relying on legacy configurations with RSA-512 or 1024-bit keys believed their traffic was secure — it wasn't. Modern attackers can factor these keys offline and decrypt captured sessions retroactively, making past traffic vulnerable even after a patch.

## Key facts
- NIST recommends RSA/DSA keys ≥ 2048 bits and symmetric keys ≥ 128 bits as current minimums for adequate security
- 56-bit DES keys can be brute-forced in under 24 hours with commodity hardware — this is a textbook CWE-326 instance
- Elliptic Curve keys require shorter lengths for equivalent strength: 256-bit ECC ≈ 3072-bit RSA
- CWE-326 differs from CWE-327 (broken algorithm) — here the *algorithm* may be sound (e.g., AES), but the *key size* (e.g., AES-40) destroys the security guarantee
- TLS 1.3 explicitly removes support for export-grade cipher suites (40/56-bit keys), directly addressing this weakness class
