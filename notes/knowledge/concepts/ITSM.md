# ITSM

## What it is
Think of ITSM as the operating manual for a hospital — every procedure, from admitting patients to discharging them, follows documented protocols so nothing falls through the cracks. IT Service Management (ITSM) is the structured set of policies, processes, and procedures organizations use to design, deliver, manage, and improve IT services across their entire lifecycle. Frameworks like ITIL provide the standardized playbook most enterprises follow.

## Why it matters
In the 2020 SolarWinds breach, attackers leveraged weaknesses in change management — a core ITSM process — to push malicious updates through the software supply chain without triggering alarms. A mature ITSM change management process requires peer review, testing, and rollback plans before any production change is approved, which would have created detection opportunities. Organizations with rigid ITSM controls are far more likely to catch unauthorized changes before they become catastrophic incidents.

## Key facts
- **Change Management** (an ITSM process) is directly tested on Security+ — unauthorized changes are a key insider threat indicator
- **Incident Management** vs. **Problem Management**: Incident = restore service fast; Problem = find root cause permanently — both are ITSM disciplines
- ITSM integrates with **CMDB (Configuration Management Database)**, which tracks all IT assets and their relationships — critical for impact analysis during breaches
- **Service Level Agreements (SLAs)** defined in ITSM set the baseline for acceptable security response times (e.g., critical vulnerability patched within 24 hours)
- ITIL 4, the current ITSM framework version, emphasizes the **Service Value System (SVS)**, aligning IT services directly to business security outcomes
