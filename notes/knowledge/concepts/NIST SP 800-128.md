# NIST SP 800-128

## What it is
Like a mechanic's maintenance log that tracks every oil change, part replacement, and inspection to keep a car roadworthy, NIST SP 800-128 provides the framework for continuously maintaining the security posture of information systems throughout their operational life. Formally titled *Guide for Security-Focused Configuration Management of Information Systems*, it defines how organizations should manage, control, and document changes to system configurations to preserve security baselines established during initial authorization.

## Why it matters
In 2020, the SolarWinds breach succeeded partly because a malicious software update slipped through without proper change control — no one verified the new build against a trusted security baseline. NIST SP 800-128 directly addresses this gap: had rigorous configuration change management been enforced, the unauthorized modification to the Orion build process would have triggered a deviation alert before deployment.

## Key facts
- Defines four core activities: **Planning**, **Identifying and Implementing Configurations**, **Controlling Configuration Changes**, and **Monitoring** — forming a continuous cycle
- Works in tandem with **NIST SP 800-53** (security controls) and **NIST SP 800-37** (Risk Management Framework); 800-128 handles the *operational maintenance* phase
- Emphasizes maintaining a **Configuration Control Board (CCB)** to review and approve all proposed changes to authorized baselines
- The **Security Impact Analysis (SIA)** is a mandatory step before any change is implemented — you must assess what could break or degrade before touching the system
- Closely tied to the **continuous monitoring** strategy; configuration drift detected during monitoring feeds back into the change control process
