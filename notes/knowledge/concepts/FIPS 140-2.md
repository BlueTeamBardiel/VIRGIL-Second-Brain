# FIPS 140-2

## What it is
Think of it as a UL certification sticker on a power strip — it doesn't tell you *what* the device does, but it proves the manufacturer built it to survive rigorous safety testing. FIPS 140-2 (Federal Information Processing Standard 140-2) is a U.S. government standard published by NIST that defines security requirements for cryptographic modules — hardware, software, or firmware that performs encryption operations. It establishes four ascending security levels (1–4) that organizations must meet to handle sensitive but unclassified federal data.

## Why it matters
When the VA (Veterans Affairs) suffered its infamous 2006 data breach exposing 26 million veterans' records on a stolen laptop, one failure point was unencrypted storage. Had FIPS 140-2 validated encryption modules been mandated and enforced on that device, the stolen data would have been cryptographically protected — making the physical theft operationally useless to the attacker.

## Key facts
- **Four security levels**: Level 1 (basic algorithm correctness) → Level 4 (physical tamper-resistance with self-destruct mechanisms)
- **Mandatory for federal agencies**: Any U.S. federal system processing sensitive unclassified information (CUI) must use FIPS 140-2 validated modules; this extends to federal contractors
- **Validates the *module*, not the algorithm**: AES is strong, but only an AES *implementation* with a FIPS certificate number is compliant — "AES-256 used" ≠ FIPS compliant
- **FIPS 140-3** (based on ISO/IEC 19790) began superseding 140-2 in September 2022; 140-2 certificates remained accepted through September 2026
- **Common caveat**: Using Windows in "FIPS mode" disables non-compliant algorithms system-wide but doesn't itself constitute full FIPS 140-2 compliance
