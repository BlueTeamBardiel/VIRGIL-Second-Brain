# CWE-407

## What it is
Like a bouncer who spends 10 minutes checking every ID while a line of thousands forms outside, inefficient algorithmic complexity lets attackers craft special inputs that force your code to do exponentially more work than expected. CWE-407 describes a vulnerability where an algorithm's worst-case performance can be triggered by an attacker, causing resource exhaustion through carefully chosen inputs rather than sheer volume.

## Why it matters
ReDoS (Regular Expression Denial of Service) attacks exploit this directly — a malicious user submits a crafted string to a web application's input validation regex, causing catastrophic backtracking that pegs CPU at 100% for seconds or minutes per request. Cloudflare suffered a global outage in 2019 when a single poorly written regex in their WAF matched against traffic in O(n²) time, taking down their entire network for ~27 minutes.

## Key facts
- **ReDoS** is the most common real-world instance — evil regex patterns like `(a+)+` exhibit exponential backtracking on inputs like `aaaaaaaaab`
- Complexity attacks target **time** (CPU exhaustion) or **space** (memory exhaustion), both leading to DoS without traditional flooding
- **Hash collision attacks** (HashDoS) exploit poor hash table implementations — crafting inputs that all map to the same bucket degrades O(1) lookups to O(n²)
- Mitigation strategies include: input length limits, regex timeout enforcement, use of linear-time regex engines (RE2, Hyperscan), and randomized hash seeds
- CVSS scoring often rates these as **Medium severity**, but business impact can be High when they hit authentication or rate-limiting code paths
