# RFC 3768

## What it is
Like a backup quarterback who automatically steps in when the starter goes down — without anyone calling a timeout — VRRP lets a standby router silently take over when the primary fails. RFC 3768 defines the Virtual Router Redundancy Protocol (VRRP), a network protocol that creates a virtual IP address shared among a group of routers, ensuring default gateway availability without manual intervention.

## Why it matters
An attacker on a local network segment can craft malicious VRRP advertisement packets with a higher priority value (up to 255) to become the elected "master" router, effectively performing a man-in-the-middle attack by redirecting all default gateway traffic through their machine. This VRRP hijacking attack requires no authentication in the base RFC 3768 specification, making it a practical threat on misconfigured enterprise networks where analysts must monitor for unexpected VRRP master transitions.

## Key facts
- VRRP uses multicast address **224.0.0.18** and IP protocol number **112** for advertisement messages
- The router with the **highest priority value (1–255)** becomes master; priority **255** is reserved for the IP address owner
- RFC 3768 includes **no mandatory authentication** — the authentication fields were deprecated from RFC 2338, making spoofing trivial without additional controls
- **Preemption is enabled by default**: a higher-priority router that comes online will automatically reclaim master status, which attackers exploit by advertising priority 255
- Replaced by **RFC 5798** (VRRPv3), which adds IPv6 support but still relies on external mechanisms (like IPsec) for authentication
