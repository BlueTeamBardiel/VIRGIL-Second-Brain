# nRF54H20

## What it is
Think of it as a Swiss Army knife with a locked vault built in — a single chip that handles both wireless radio communications AND stores your most sensitive secrets in a hardware-isolated safe. The nRF54H20 is Nordic Semiconductor's multi-core SoC (System-on-Chip) featuring dedicated application, network, and security processing cores, with an integrated PSA-certified hardware security domain containing secure key storage, cryptographic accelerators, and a hardware root of trust.

## Why it matters
In IoT device security, firmware extraction attacks are common — an attacker dumps flash memory via JTAG or SWD debug interfaces to steal firmware, API keys, or cryptographic material. The nRF54H20's security domain enforces hardware-level isolation so that even a fully compromised application core cannot access secrets stored in the secure domain, meaning a supply-chain attacker who backdoors your application firmware still cannot exfiltrate the device's private keys used for attestation or TLS mutual authentication.

## Key facts
- Contains a dedicated **CRACEN** (Cryptographic Accelerator and Root of trust for Nordic) hardware security subsystem, providing AES-256, ECDH, ECDSA, and SHA-3
- Implements **ARM TrustZone** across cores, enforcing Secure/Non-Secure memory and peripheral partitioning
- Achieves **PSA Certified Level 2** — relevant for IoT product security compliance frameworks and supply chain risk assessments
- Debug access (SWD/JTAG) can be **permanently or temporarily locked** via lifecycle state management, preventing runtime debug-based attacks post-production
- Supports **Secure Boot with hardware root of trust** — each firmware image must chain back to a burned OTP key, defeating unsigned firmware injection
