# T1053

## What it is
Like setting a recurring alarm on your phone to remind you to do something later — except the "alarm" runs malicious code and *you* didn't set it. T1053 (Scheduled Task/Job) is a MITRE ATT&CK technique where adversaries abuse OS scheduling mechanisms (Windows Task Scheduler, cron, systemd timers) to execute malicious payloads automatically at defined times or intervals.

## Why it matters
The TrickBot banking trojan famously used Windows Scheduled Tasks to maintain persistence across reboots — even if the original dropper was deleted, the scheduled task would re-execute the payload on login. Defenders hunting for this look for tasks created by unusual parent processes (e.g., `cmd.exe` spawning `schtasks.exe`) or tasks pointing to temp directories like `%AppData%` or `%Temp%`.

## Key facts
- **Sub-techniques include:** T1053.002 (At), T1053.003 (Cron), T1053.005 (Scheduled Task - Windows), T1053.006 (systemd Timers), T1053.007 (Container Orchestration Jobs)
- **Dual purpose:** Used for both **Persistence** and **Privilege Escalation** in the MITRE ATT&CK framework
- **Detection artifact:** On Windows, Event ID **4698** (task created) and **4702** (task modified) are key log sources for hunting malicious scheduled tasks
- **Living-off-the-land:** Attackers use the native `schtasks.exe` or `at.exe` binaries, making detection harder since no new tools are dropped
- **Privilege escalation vector:** A task running as SYSTEM that executes a user-writable binary is a classic privilege escalation path (DLL hijacking + scheduled task)
