# T1059.002

## What it is
Like hiring a native speaker to deliver a suspicious message so it sounds completely legitimate, AppleScript lets attackers use macOS's own built-in automation language to issue commands that appear trustworthy to the system. It is a MITRE ATT&CK sub-technique where adversaries abuse AppleScript — macOS's native scripting language — to execute malicious code, automate GUI interactions, and communicate with applications via Apple Events.

## Why it matters
In the OSX.Dok malware campaign, attackers used AppleScript to silently install a rogue root certificate and redirect all traffic through an attacker-controlled proxy, bypassing HTTPS protections entirely. Because AppleScript is a legitimate macOS feature signed by Apple itself, traditional signature-based AV tools often miss it — making it a favored living-off-the-land technique on Mac endpoints.

## Key facts
- AppleScript can send **Apple Events** to any scriptable application, including Finder, Terminal, and browsers — giving attackers broad system reach without dropping a binary
- Attackers commonly deliver malicious `.scpt` or `.applescript` files disguised as productivity macros inside Office documents
- AppleScript can **bypass Gatekeeper** when executed through a trusted parent process like Microsoft Word or a legitimate app
- Detection focus: monitor for `osascript` process executions (the AppleScript runtime), especially with `-e` inline command flags, which indicate scripted one-liners
- This technique pairs frequently with **T1566.001 (Spearphishing Attachment)** as the initial delivery vector on macOS targets
