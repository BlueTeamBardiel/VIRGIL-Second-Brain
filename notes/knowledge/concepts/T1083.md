# T1083

## What it is
Like a burglar casing a house by peering through windows to map out where the valuables are kept, File and Directory Discovery is the act of an attacker systematically enumerating files and folders on a compromised system. Precisely: T1083 describes adversary behavior where they list directory contents, search for specific file types, or locate sensitive files (credentials, configs, backups) to understand the target environment and plan next moves.

## Why it matters
During the 2020 SolarWinds intrusion, SUNBURST malware performed file and directory discovery to identify security tools and forensic software before proceeding — if it detected certain filenames associated with analysis environments, it halted activity entirely. Defenders who monitor for abnormal enumeration commands (especially from unusual parent processes) can catch attackers in this reconnaissance phase before data exfiltration begins.

## Key facts
- Common commands used: `dir`, `tree`, `ls`, `find`, `Get-ChildItem` (PowerShell) — any unexpected execution of these by non-interactive processes is a red flag
- Attackers specifically hunt for files named `passwords.txt`, `.ssh/`, `web.config`, `*.kdbx` (KeePass databases), and backup archives
- This technique appears in the **Discovery** tactic (TA0007) of the MITRE ATT&CK framework
- Legitimate admin tools (like Everything.exe or built-in OS commands) are frequently abused, making detection context-dependent rather than signature-based
- Effective detection relies on baselining normal enumeration behavior and alerting on high-volume or unusual-path directory traversals from scripting hosts like `powershell.exe` or `cmd.exe`
