# CWE-416

## What it is
Like a hotel that re-rents a room before housekeeping clears out the previous guest's belongings — someone else can now access private items left behind. Use-After-Free (UAF) is a memory corruption vulnerability where a program continues to reference a memory location after that memory has been freed/deallocated, allowing an attacker to control what occupies that memory space next.

## Why it matters
The 2021 Chrome V8 engine exploit (CVE-2021-21224) was a UAF vulnerability that allowed attackers to achieve remote code execution simply by luring victims to a malicious webpage. Defenders patched it by adding heap isolation techniques and pointer sanitization, demonstrating that UAF bugs in browser engines are prime targets for drive-by download attacks.

## Key facts
- UAF bugs consistently rank in the **CWE Top 25 Most Dangerous Software Weaknesses** and are heavily targeted in browser and kernel exploits
- The attack pattern: **allocate → free → reallocate with attacker-controlled data → trigger dangling pointer** — the freed chunk is reclaimed and poisoned
- Exploitation often leverages **heap spraying** to predictably place malicious data in the freed memory region before the dangling pointer is dereferenced
- Modern mitigations include **MemorySanitizer**, **AddressSanitizer (ASan)**, heap quarantine, and pointer authentication codes (PAC) on ARM
- UAF in kernel space (Ring 0) is especially severe — it can lead to **privilege escalation** by overwriting function pointers in kernel objects (e.g., Linux `sock` structures)
