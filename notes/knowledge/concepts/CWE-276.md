# CWE-276

## What it is
Like a new apartment where the landlord forgets to change the locks after construction — leaving skeleton keys accessible to anyone who walks by — CWE-276 occurs when software is installed with insecure default permissions that grant excessive access to unintended users. Specifically, it describes files, directories, or other resources left world-readable or world-writable because the developer never restricted access during initial setup.

## Why it matters
In 2021, misconfigured default permissions on Apache HTTP Server installations left configuration files readable by all local users, allowing low-privileged attackers to harvest credentials stored in plaintext config files and escalate privileges. Defenders mitigate this by applying the principle of least privilege at install time — running hardening scripts or CIS Benchmarks immediately after deployment to reset permissions to their secure baseline.

## Key facts
- Default permissions of `0777` (rwxrwxrwx on Linux) or "Everyone: Full Control" on Windows are classic CWE-276 signatures
- Affects installation packages, containers, and cloud storage buckets — S3 buckets left publicly readable are a modern incarnation
- Distinguishable from CWE-732 (Incorrect Permission Assignment) — 276 is specifically about **installation-time** defaults, not runtime misconfiguration
- CVSS scoring for these vulnerabilities often ranges 5.5–7.8 depending on what data is exposed or whether execution is possible
- Remediation involves `chmod`/`chown` hardening, `umask` settings in service startup scripts, and automated compliance scanning tools like Lynis or OpenSCAP
