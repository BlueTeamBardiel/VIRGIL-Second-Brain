# T1059.011

## What it is
Like a master key that opens doors the locksmith never intended, Lua scripting lets attackers embed malicious logic inside applications that already trust the Lua interpreter. Lua is a lightweight scripting language frequently embedded in games, network devices, and security tools; adversaries abuse it to execute arbitrary commands within the context of a host application that has already been granted elevated trust or permissions.

## Why it matters
In 2022, threat actors exploited Lua scripting capabilities embedded in legitimate network monitoring software to execute payloads without spawning obvious child processes like `cmd.exe` or `powershell.exe`, effectively bypassing endpoint detection rules tuned for traditional script interpreters. This demonstrates why defenders must monitor *application-level* scripting, not just OS-level shells — the attack surface is hiding inside trusted software.

## Key facts
- Lua is commonly embedded in products like **Nmap** (NSE scripts), **Redis**, **World of Warcraft**, and various firewalls — all potential abuse vectors
- Because Lua runs *inside* a host process, malicious activity may appear as benign parent-process behavior, defeating parent-child process monitoring
- Attackers use Lua's `os.execute()` and `io.popen()` functions to run OS commands from within the interpreter
- Detection requires monitoring for **unusual Lua script loading**, unexpected network connections from Lua-capable applications, and script content inspection
- MITRE categorizes this under Command and Scripting Interpreter (T1059) — understanding the parent technique is required for CySA+ context
