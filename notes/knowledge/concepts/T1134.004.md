# T1134.004

## What it is
Like a con artist who steals a celebrity's passport to pass airport security as someone famous, this technique involves duplicating an existing access token to impersonate another user or process. T1134.004 (Token Impersonation/Theft - Parent PID Spoofing) specifically refers to an attacker spawning a new process with a *spoofed* parent process ID, making malicious child processes appear to be legitimate children of trusted system processes like `explorer.exe` or `svchost.exe`.

## Why it matters
During the SolarWinds attack, adversaries used parent PID spoofing to make malicious processes appear as children of legitimate Windows services, evading EDR tools that use process lineage as a detection signal. Defenders who rely solely on parent-child process relationships without cryptographic attestation will miss these attacks entirely.

## Key facts
- Windows `CreateProcess()` API with `PROC_THREAD_ATTRIBUTE_PARENT_PROCESS` attribute allows specifying an arbitrary parent PID — this is the primary mechanism exploited
- Tools like `Cobalt Strike`, `Metasploit`, and `psgetsystem` implement parent PID spoofing natively
- Spoofed parent PIDs break behavioral analytics that model normal process trees (e.g., `Word.exe` should never spawn `cmd.exe`)
- This technique requires a handle to the target parent process, meaning the attacker generally needs `PROCESS_CREATE_PROCESS` access rights on the target
- Detection relies on comparing the *reported* parent PID in process creation events (Event ID 4688) against the *actual* creator PID using ETW (Event Tracing for Windows)
