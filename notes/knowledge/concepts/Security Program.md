# Security Program

## What it is
Like a city's entire public safety infrastructure — police, fire, building codes, emergency services — working together rather than a single lock on a single door, a security program is the comprehensive, organization-wide framework that coordinates people, processes, and technology to protect information assets. It encompasses governance, risk management, policies, controls, and continuous improvement under unified leadership, typically a CISO.

## Why it matters
When Target suffered its 2013 breach (40 million cards compromised), investigators found security tools *were* alerting — but no cohesive program existed to act on those alerts. A mature security program with defined incident response workflows and clear ownership would have caught the HVAC vendor intrusion before attackers reached the POS systems. The breach cost Target over $200 million and demonstrated that technology alone without programmatic governance fails.

## Key facts
- A security program is built on a framework such as **NIST CSF**, **ISO 27001**, or **NIST RMF**, which provides the structural backbone for all controls and policies
- The **Plan-Do-Check-Act (PDCA)** cycle is the operational engine of a security program, driving continuous improvement rather than one-time implementation
- Effective programs require **executive sponsorship** (often a CISO reporting to the CEO or Board) to ensure security decisions carry organizational authority
- A security program must include **metrics and KPIs** (e.g., mean time to detect, patch compliance rates) to demonstrate measurable risk reduction
- **Security awareness training** is a required programmatic component under frameworks like NIST 800-53 and is explicitly tested on Security+ and CySA+ as a control family
