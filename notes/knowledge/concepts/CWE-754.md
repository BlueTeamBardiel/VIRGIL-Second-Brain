# CWE-754

## What it is
Like a pilot who skips post-flight checks because "the flight felt fine," this weakness describes code that fails to test return values or status codes after critical operations. CWE-754 (Improper Check for Unusual or Exceptional Conditions) occurs when software doesn't validate whether a function succeeded, failed, or returned an anomalous result before proceeding with execution.

## Why it matters
In 2014, the Apple "goto fail" SSL bug (CVE-2014-1266) was a catastrophic real-world example — a duplicated `goto fail` line caused signature verification to be skipped entirely, returning success regardless of the actual cryptographic check result. An attacker performing a man-in-the-middle attack could present a forged certificate that iOS and OS X would silently accept as valid, completely bypassing TLS authentication.

## Key facts
- Unchecked return values from memory allocation functions (e.g., `malloc()`) can cause null pointer dereferences that lead to crashes or code execution
- This weakness commonly enables **authentication bypasses** — if a login check's return value is ignored, execution continues as if authentication succeeded
- CVSS scores for CWE-754 vulnerabilities often range high (7.0–9.8) because they frequently undermine security-critical controls
- Appears frequently in C/C++ code but also affects Java, Python, and other languages when exception handling is incomplete or swallowed silently
- Mitigations include static analysis tools (SAST), compiler warnings for ignored return values (`-Wunused-result` in GCC), and mandatory code review checklists for security-sensitive functions
