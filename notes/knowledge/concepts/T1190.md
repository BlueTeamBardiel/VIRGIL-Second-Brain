# T1190

## What it is
Like a burglar who doesn't pick the lock but instead walks through a door that was already broken by the manufacturer, Exploit Public-Facing Application (T1190) is the technique where attackers exploit vulnerabilities in internet-exposed services — web apps, VPNs, firewalls — to gain initial access without needing credentials.

## Why it matters
In 2021, attackers exploited ProxyLogon (CVE-2021-26855) in Microsoft Exchange servers worldwide, using a pre-authentication SSRF vulnerability to bypass authentication entirely and deploy web shells — all before any patch could be applied at scale. Defenders who had network segmentation and WAFs in place dramatically reduced their blast radius compared to those running fully exposed Exchange instances.

## Key facts
- **Initial Access technique** (MITRE ATT&CK Tactic: TA0001) — this is how adversaries get their foot in the door, not how they move laterally afterward
- Common targets include web applications, VPN gateways (Pulse Secure, Citrix), firewalls, and internet-facing databases
- Exploits may be pre-authentication (no credentials needed) or post-authentication; pre-auth vulns are far more dangerous and command higher CVE severity scores
- Key mitigations: aggressive patch management, web application firewalls (WAF), attack surface reduction (don't expose what doesn't need to be public), and vulnerability scanning with tools like Shodan monitoring
- Distinguished from T1133 (External Remote Services) — T1190 requires exploiting a *vulnerability*, whereas T1133 abuses *legitimate* remote access functionality
