# T1078.003

## What it is
Like a burglar who steals a master key from the front desk instead of picking the lock, attackers abuse **local accounts** — built-in or manually created user accounts that exist only on a single machine — to gain or maintain access without touching domain infrastructure. These accounts (think `Administrator` on Windows or `root` on Linux) authenticate locally and bypass centralized identity controls entirely.

## Why it matters
During the 2021 Colonial Pipeline ransomware attack, attackers leveraged a compromised VPN account tied to a legacy local credential — no domain authentication required, no centralized logging triggered. Defenders who only monitor Active Directory events completely missed the lateral movement because local account activity often falls outside SIEM scope.

## Key facts
- The built-in Windows `Administrator` account (RID 500) cannot be locked out by default, making it a high-value target even when renamed
- Local accounts used across multiple machines with the **same password** enable lateral movement at scale — a technique countered directly by Microsoft's LAPS (Local Administrator Password Solution)
- Local account logon events generate **Event ID 4624** with Logon Type 2 (interactive) or Type 3 (network); Type 10 indicates remote interactive (RDP)
- CIS Benchmark and DISA STIG both recommend disabling the built-in Administrator account and enforcing unique local passwords per host
- Detection signature: local admin account authenticating **over the network** (especially via SMB/PsExec) is anomalous and a strong lateral movement indicator
