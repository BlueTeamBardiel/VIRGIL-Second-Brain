# CWE-400

## What it is
Like a prankster who keeps calling a restaurant to place orders they never pick up, eventually the kitchen grinds to a halt serving no real customers — this is uncontrolled resource consumption. CWE-400 occurs when software fails to limit how much CPU, memory, disk, or network bandwidth a single user or process can consume, enabling attackers to exhaust those resources and deny service to legitimate users.

## Why it matters
The 2016 Mirai botnet DDoS attacks exploited exactly this weakness — by flooding targets with traffic, they overwhelmed servers' ability to allocate connection resources, taking down major DNS provider Dyn and making sites like Twitter and Netflix unreachable for hours. Defenders counter this with rate limiting, connection throttling, and resource quotas at the application and network layer.

## Key facts
- CWE-400 is the root weakness behind many **Denial of Service (DoS)** and **Distributed DoS (DDoS)** attacks, including slowloris HTTP attacks that hold connections open indefinitely
- It differs from CWE-770 (Allocation Without Limits) in that CWE-400 focuses on **missing controls**, while CWE-770 focuses on the allocation call itself
- Common vulnerable resources: **thread pools, database connections, file descriptors, memory buffers, and CPU cycles**
- Mitigations include: input validation on loop bounds, enforcing **timeouts and maximum request sizes**, and implementing backpressure mechanisms
- Regex patterns that cause catastrophic backtracking (ReDoS) are a CPU-specific form of CWE-400 — a single malformed input string can spike a server to 100% CPU
