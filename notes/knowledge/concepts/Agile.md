# Agile

## What it is
Like a jazz band improvising together rather than following a rigid score, Agile is an iterative software development methodology that delivers working software in short cycles (called sprints) instead of one massive final release. Teams continuously adapt to changing requirements, prioritize collaboration, and release functional increments every 1–4 weeks rather than waiting for a "perfect" product at the end.

## Why it matters
In secure software development, Agile enables security teams to integrate threat modeling and code review into every sprint rather than bolting on security at the end — a practice called "shifting left." For example, a development team using Agile with integrated static analysis (SAST) can catch an SQL injection vulnerability in Sprint 3 rather than discovering it during a penetration test after deployment, dramatically reducing remediation cost and exposure window.

## Key facts
- Agile is a core component of the **Secure Software Development Lifecycle (SSDLC)** — Security+ expects you to know it as a methodology that supports iterative security review
- The **Scrum** framework is the most common Agile implementation, using sprints, daily standups, and retrospectives
- **DevSecOps** embeds security practices directly into Agile pipelines — continuous integration/continuous delivery (CI/CD) pipelines automate security testing per sprint
- Agile contrasts with **Waterfall**, which performs security testing only at the end; Agile's shorter feedback loops reduce the cost of fixing vulnerabilities by up to 6x
- The **Agile Manifesto** (2001) emphasizes individuals over processes, working software over documentation, and responding to change over following a fixed plan — all relevant to adaptive security posture
