# T1218.003

## What it is
Like borrowing a bank guard's uniform to walk past security unchallenged, attackers use **CMSTP.exe** — Microsoft's Connection Manager Profile Installer — to execute malicious code under the guise of a trusted, signed Windows binary. Specifically, adversaries craft a malicious INF file (a setup configuration) and pass it to CMSTP.exe, which then executes it with elevated privileges, bypassing both AppLocker and UAC controls.

## Why it matters
In 2018, the MuddyWater APT group leveraged CMSTP.exe to bypass UAC and execute PowerShell payloads during campaigns targeting Middle Eastern government organizations. Because CMSTP is a signed Microsoft binary, application whitelisting solutions that trust Microsoft-signed executables failed to block the execution chain, allowing full code execution with elevated privileges.

## Key facts
- CMSTP.exe is a **Living-off-the-Land Binary (LOLBin)** — a legitimate Windows tool repurposed for malicious execution
- It can bypass **UAC (User Account Control)** and **AppLocker** policies, since it's a trusted, Microsoft-signed executable
- Attackers supply a malformed **INF file** with a `RegisterOCXSection` that points to a remote scriptlet or command, which CMSTP executes automatically
- CMSTP can also be used for **COM object hijacking** and loading remote payloads via UNC paths (e.g., `\\attacker.com\payload.sct`)
- Detection focus: monitor for `cmstp.exe` spawning child processes (especially `cmd.exe`, `powershell.exe`) or making unusual network connections — both are strong indicators of abuse
