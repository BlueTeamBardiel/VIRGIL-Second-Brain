# T1564.004

## What it is
Like hiding a note inside a wall before the drywall goes up, NTFS Alternate Data Streams (ADS) let attackers store hidden data inside a legitimate file's metadata streams — invisible to standard directory listings. Precisely, ADS is a feature of the NTFS filesystem that allows multiple data streams to be attached to a single file, where only the primary stream is shown by default tools like `dir`.

## Why it matters
During the 2020s, malware families like Astaroth used ADS to hide malicious payloads inside innocuous `.txt` or `.jpg` files, launching them via `wscript.exe` to evade endpoint detection. A defender running standard `dir` commands would see a normal-looking file at its expected size, while the actual payload silently executes from the hidden stream.

## Key facts
- ADS syntax uses a colon separator: `malware.exe:hidden_payload` — the hidden stream is never visible in standard `dir` output but CAN be revealed with `dir /r`
- Streams can execute code directly: `wscript.exe C:\legit.txt:script.vbs` runs the hidden VBS without exposing it in the primary file
- ADS only works on **NTFS** volumes — copying a file to FAT32 or exFAT strips all alternate streams, which can be both a detection and containment technique
- Tools like **Sysinternals Streams.exe** and PowerShell's `Get-Item -Stream *` can enumerate hidden streams for forensic analysis
- ADS is categorized under MITRE ATT&CK **Hide Artifacts (T1564)**, sub-technique **.004**, reflecting its role in defense evasion
