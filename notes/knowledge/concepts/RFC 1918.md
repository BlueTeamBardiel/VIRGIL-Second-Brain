# RFC 1918

## What it is
Like neighborhoods that exist on a private housing development's internal map but never appear in a public city directory, RFC 1918 addresses are routable within private networks but invisible and unreachable from the public internet. RFC 1918 is the IETF standard that reserves three specific IP address ranges exclusively for private, non-routable use, preventing them from being assigned to public-facing infrastructure.

## Why it matters
During a network reconnaissance engagement, an attacker who gains a foothold inside a corporate network can use RFC 1918 addresses to map internal infrastructure that's completely hidden from external scanners — this is why lateral movement tools like Nmap are used post-exploitation to discover internal hosts. Defenders leverage RFC 1918 ranges to implement network segmentation, ensuring that sensitive internal systems like domain controllers (typically at 10.x.x.x or 192.168.x.x) never receive direct internet traffic, shrinking the attack surface dramatically.

## Key facts
- **Three reserved ranges:** 10.0.0.0/8 (Class A), 172.16.0.0/12 (Class B), 192.168.0.0/16 (Class C)
- RFC 1918 addresses **cannot be routed on the public internet** — ISPs drop packets sourced from or destined to these ranges at border routers
- Network Address Translation (NAT) is the mechanism that allows RFC 1918 hosts to communicate externally by mapping private IPs to a public IP
- Seeing RFC 1918 source addresses in internet-facing traffic is a **red flag for IP spoofing** — ingress/egress filtering (BCP 38) blocks this
- APIPA range (169.254.0.0/16) is **not** RFC 1918 — it's link-local and defined separately in RFC 3927; mixing these up is a common exam trap
