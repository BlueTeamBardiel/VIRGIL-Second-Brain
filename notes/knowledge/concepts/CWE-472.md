# CWE-472

## What it is
Like a referee who accepts a player's self-reported score without checking the scoreboard, a web application with this flaw trusts hidden form fields sent back from the client. CWE-472 (External Control of Assumed-Immutable Web Parameter) occurs when server-side code treats hidden HTML fields, URL parameters, or cookies as trustworthy constants — even though a user can trivially modify them before resubmission.

## Why it matters
In a classic e-commerce attack, a shopper intercepts a checkout form containing `<input type="hidden" name="price" value="99.99">` using Burp Suite, changes the value to `0.01`, and resubmits. If the server blindly trusts that hidden field instead of recalculating the price from its own database, the attacker purchases items for nearly nothing — a real attack pattern seen against early online retailers and still found in poorly audited applications today.

## Key facts
- Hidden HTML form fields provide **zero security**; any browser developer tool or proxy can modify them before the request is sent
- Common targets include: `price`, `discount`, `user_role`, `account_id`, `is_admin`, and session-related tokens stored client-side
- The fix is **server-side state management** — store sensitive values in the server session or database, never trust the client's copy
- Distinguished from CWE-807 (reliance on untrusted inputs in security decisions) by its specific focus on parameters *assumed* to be constant by developers
- Testing methodology: intercept requests with a proxy (Burp/ZAP), modify hidden/URL parameters, observe whether the server enforces its own authoritative values
