# CWE-331

## What it is
Imagine a lottery machine that only ever picks from 10 possible numbers instead of millions — an attacker can simply try all 10. CWE-331 is **Insufficient Entropy**, where a system generates random values (keys, tokens, nonces, session IDs) using a source that doesn't have enough unpredictability to resist brute-force guessing. The "randomness" pool is too small or too predictable, making supposedly secret values recoverable.

## Why it matters
In 2008, the Debian OpenSSL vulnerability (CVE-2008-0166) demonstrated CWE-331 catastrophically: a misguided code patch reduced the entropy in key generation to only the process ID — roughly 32,768 possible values. Attackers could pre-compute every possible SSH and SSL key that any Debian system had generated, breaking authentication across thousands of production servers worldwide with a simple lookup table.

## Key facts
- Entropy is measured in bits; a 128-bit key space requires a source capable of producing 2¹²⁸ unpredictable outcomes — seeding from system time alone provides nowhere near that
- CWE-331 frequently appears when developers use `rand()` or `Math.random()` (PRNGs) instead of cryptographically secure alternatives like `/dev/urandom`, `CryptGenRandom`, or `CSPRNG` APIs
- Session tokens with insufficient entropy enable **session prediction attacks** — an attacker enumerates valid tokens rather than stealing them
- NIST SP 800-90A defines approved Deterministic Random Bit Generators (DRBGs); using non-approved sources violates this standard and fails compliance audits (PCI-DSS, FIPS 140-2)
- Entropy starvation can occur in virtualized/embedded environments where hardware random sources are unavailable, making VM-hosted key generation silently weaker
