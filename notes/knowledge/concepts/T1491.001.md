# T1491.001

## What it is
Like a graffiti artist tagging the front window of a store to embarrass the owner, website defacement is when an attacker replaces or alters the visible content of a public-facing website. It is a sub-technique of Defacement (T1491) specifically targeting internal or external websites to deliver messaging, damage reputation, or signal a successful compromise.

## Why it matters
During the 2021 Ukraine government website attacks, threat actors defaced multiple Ukrainian ministry sites with threatening messages before the destructive WhisperGate malware was deployed — using defacement as both psychological warfare and a distraction. Defenders who monitor for unauthorized file changes to web roots (e.g., index.html modifications) via file integrity monitoring can catch this early before deeper damage occurs.

## Key facts
- Attackers typically gain access first via SQLi, RFI, stolen credentials, or CMS vulnerabilities (WordPress, Drupal) before modifying web content
- Common motivations include hacktivism, nation-state messaging, and proving access — it is often *not* financially motivated
- File Integrity Monitoring (FIM) tools like Tripwire or OSSEC are primary detective controls against defacement
- Web Application Firewalls (WAFs) serve as preventive controls by blocking common exploit vectors used to gain initial access
- Defacement is classified under the **Impact** tactic in the MITRE ATT&CK framework, meaning the attacker has already achieved significant access before executing it
