# T1105

## What it is
Like a smuggler tossing tools over a prison wall after the initial break-in, attackers use Ingress Tool Transfer to pull additional malware, scripts, or utilities into a compromised system from an external source. Precisely: T1105 describes the technique of transferring files from an external system into a compromised environment using tools already present on the victim host (e.g., curl, wget, certutil, PowerShell's `Invoke-WebRequest`).

## Why it matters
During the SolarWinds supply chain attack, after initial access was established, SUNBURST malware used HTTP-based callbacks to download secondary payloads from attacker-controlled C2 infrastructure — a textbook T1105 execution. Defenders who monitor for unexpected outbound connections from servers pulling executable content can catch this stage before the attacker fully establishes their toolkit.

## Key facts
- Commonly abused Living-off-the-Land Binaries (LOLBins): `certutil -urlcache`, `bitsadmin /transfer`, `curl`, `wget`, `PowerShell Invoke-WebRequest`
- Often occurs in the **post-exploitation** phase — after initial access, before lateral movement
- Defenders should alert on processes making external network calls that don't normally do so (e.g., `certutil.exe` reaching out to the internet)
- HTTPS is frequently used to blend transfer traffic with normal web traffic and evade inspection
- Detection data sources per MITRE: network traffic content, file creation events, and process command-line parameters
