# T1110

## What it is
Like a locksmith trying every key on a giant ring until one fits, brute force attacks systematically attempt credentials until authentication succeeds. T1110 is MITRE ATT&CK's technique covering all credential brute forcing methods — password spraying, credential stuffing, dictionary attacks, and raw brute force — against services like SSH, RDP, and web applications.

## Why it matters
In the 2021 Colonial Pipeline attack, threat actors gained initial access through a compromised VPN password — likely obtained via credential stuffing (T1110.004), where previously breached username/password pairs from other sites were tried against the VPN portal. Had multi-factor authentication been enabled on that VPN, the single-factor brute force vector would have been neutralized entirely.

## Key facts
- **Four sub-techniques**: T1110.001 (Password Guessing), T1110.002 (Password Cracking), T1110.003 (Password Spraying), T1110.004 (Credential Stuffing)
- **Password spraying** uses one common password (e.g., "Summer2024!") across many accounts to avoid account lockout thresholds — a favorite against Active Directory environments
- **Credential stuffing** exploits password reuse; tools like Sentry MBA and Snipr automate this against login APIs
- **Detection signatures** include repeated failed logins (Event ID 4625 on Windows), sudden spikes in authentication requests, and login attempts from unusual geographies or IPs
- **Mitigations**: account lockout policies, MFA, CAPTCHA, rate limiting, and monitoring for distributed low-volume spraying that evades lockout
