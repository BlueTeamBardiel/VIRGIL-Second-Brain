# NetFlow

## What it is
Think of NetFlow as the phone company's call detail records — it doesn't record *what* you said, but it logs *who called whom, for how long, and how often*. NetFlow is a Cisco-developed network protocol that captures metadata about IP traffic flows: source/destination IP, ports, protocols, byte counts, and timestamps. It provides traffic visibility without the storage overhead of full packet capture.

## Why it matters
When a threat hunter suspects data exfiltration, full PCAP logs may be unavailable or prohibitively large — but NetFlow data can reveal that an internal host transferred 4GB to an unusual external IP at 2 AM over port 443, a pattern invisible in firewall logs alone. Security teams use NetFlow analysis tools like SolarWinds NTA or ntopng to baseline normal traffic behavior and flag anomalies like beaconing malware or lateral movement between workstations.

## Key facts
- NetFlow records are generated by routers/switches acting as **flow exporters** and sent to a **flow collector** for analysis — the collector does the heavy lifting, not the network device
- A "flow" is defined by the **5-tuple**: source IP, destination IP, source port, destination port, and protocol
- NetFlow is **not full packet capture** — it captures metadata only, making storage efficient but useless for payload inspection
- **IPFIX** (IP Flow Information Export) is the IETF-standardized evolution of NetFlow v9, used interchangeably in many modern tools
- NetFlow analysis is a core technique in **Network Traffic Analysis (NTA)** and is explicitly tested in CySA+ as a method for detecting C2 beaconing and insider threats
