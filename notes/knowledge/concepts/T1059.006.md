# T1059.006

## What it is
Like using the building's own master key to unlock every door, Python gives attackers a universal tool already trusted by the operating system. T1059.006 describes adversaries executing malicious commands or scripts using Python interpreters natively present on macOS, Linux, or Windows systems. Because Python is a legitimate administrative tool, its activity often blends seamlessly into normal operational noise.

## Why it matters
During the 2020 SolarWinds campaign and subsequent intrusions, threat actors used Python scripts to automate lateral movement and data exfiltration because Python was already installed on victim servers. A defender monitoring for unusual PowerShell activity might miss identical behavior wrapped in a Python subprocess call — making detection ruleset parity across scripting engines critical for any SOC.

## Key facts
- Python is pre-installed on most macOS and Linux systems, making it a **living-off-the-land** technique requiring zero additional tooling
- Attackers commonly use `python -c "import socket; ..."` one-liners to spawn reverse shells without writing files to disk (fileless execution)
- Python scripts can import OS modules (`os`, `subprocess`, `ctypes`) to execute system commands, spawn processes, or inject shellcode
- Detection focus: monitor for Python spawning unusual child processes, making network connections, or running as SYSTEM/root without user context
- Defenders should audit environments using **application whitelisting** (e.g., AppLocker, WDAC) to restrict which scripts Python is permitted to execute
