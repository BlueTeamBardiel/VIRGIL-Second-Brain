# CWE-204

## What it is
Like a bouncer who says "Wrong password" versus "That username doesn't exist" — each response leaks a different secret. CWE-204 is Observable Response Discrepancy, where a system returns distinguishably different responses to valid versus invalid inputs, allowing an attacker to enumerate hidden information. The difference can appear in HTTP status codes, error messages, response timing, or page content.

## Why it matters
In a classic user enumeration attack against a login form, an attacker submits hundreds of usernames: valid accounts return "Incorrect password" while invalid ones return "Account not found." This lets the attacker build a confirmed list of real usernames before even attempting to crack passwords — dramatically narrowing a credential stuffing attack. The 2012 LinkedIn breach exploitation and countless phishing campaigns relied on exactly this reconnaissance technique.

## Key facts
- **Observable discrepancy types** include: different HTTP status codes (200 vs. 404), different error message text, different response body lengths, or measurable timing differences
- **User enumeration** is the most common exploitation of CWE-204 — attackers confirm valid usernames/emails before launching brute-force or credential stuffing attacks
- **Mitigation**: Return identical responses (same message, same HTTP code, same timing) for both valid and invalid inputs — often called a "generic error response"
- **Timing attacks** are a subtle subtype: even if messages match, a database lookup for a real user may take longer than rejecting a fake one, leaking information through response time
- **OWASP** lists user enumeration as a component of authentication failures (formerly OWASP A2), making CWE-204 directly relevant to authentication hardening assessments
