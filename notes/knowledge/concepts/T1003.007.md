# T1003.007

## What it is
Like reading a restaurant's recipe book by looking at the scraps in the trash rather than asking the chef, this technique extracts credentials by dumping the memory of the `/proc` filesystem on Linux systems. Specifically, adversaries read `/proc/[pid]/mem` or `/proc/[pid]/maps` to scrape plaintext passwords from the memory space of processes like `sshd`, `gnome-keyring`, or `sudo`.

## Why it matters
In the 2020 compromise of cloud-hosted Linux servers, attackers used `mimipenguin` and custom scripts targeting `/proc` memory to harvest SSH credentials from running sessions without ever touching `/etc/shadow`. This allowed lateral movement across the environment using legitimate credentials, bypassing all password-hashing protections entirely.

## Key facts
- `/proc/[pid]/mem` is a pseudo-file that maps a process's virtual memory — readable by root or the process owner, making privilege escalation a prerequisite
- Tools like **mimipenguin**, **LaZagne**, and **mimipy** automate this technique on Linux/Unix systems
- Unlike `/etc/shadow` (which stores hashed passwords), process memory can contain **plaintext credentials** that applications temporarily hold during authentication
- Defenders can detect this via auditd rules monitoring `open()` syscalls on `/proc/*/mem` with rules like `-a always,exit -F arch=b64 -S ptrace`
- Mitigation includes enabling **Yama LSM** (`/proc/sys/kernel/yama/ptrace_scope = 2`) to restrict cross-process memory reads
