# CWE-202

## What it is
Like a bank employee who whispers your account balance loud enough for the waiting room to hear while trying to tell *you* it's wrong — sensitive information gets exposed precisely during the process of handling errors or exceptions. CWE-202 is the unintentional exposure of sensitive data through informational disclosures in error messages, debug output, or system responses during exceptional conditions.

## Why it matters
During the 2012 reconnaissance phase against numerous financial institutions, attackers deliberately sent malformed inputs to web applications to trigger verbose stack traces, revealing internal IP addresses, database schemas, and software versions. That harvested intelligence directly enabled subsequent targeted exploitation — turning a "free" error message into a roadmap for attack.

## Key facts
- Error messages revealing stack traces, database query strings, or file paths are classic CWE-202 manifestations
- OWASP lists information exposure through error messages under A05 (Security Misconfiguration) in their Top 10
- The fix follows the "fail silently, log internally" principle: show generic errors to users, write detailed diagnostics only to protected server-side logs
- Differs from CWE-209 (Error Message Information Exposure) in scope — CWE-202 covers *any* sensitive data leaked during exception handling, not just error message content
- Common in Java/.NET applications where unhandled exceptions propagate raw exception objects directly to HTTP responses
