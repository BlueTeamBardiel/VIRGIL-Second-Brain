# CWE-287

## What it is
Like a bouncer who waves someone in because they're *wearing* a VIP wristband — without checking whether it's forged — improper authentication means the system accepts proof of identity without genuinely verifying it. CWE-287 covers any weakness where a system fails to adequately prove that a claimed identity is actually legitimate. This includes skipped verification steps, trusting unvalidated tokens, or relying on client-supplied data to confirm who someone is.

## Why it matters
In 2022, attackers exploited improper authentication in Cisco's SSMS licensing portal by manipulating HTTP headers the server blindly trusted — granting admin access without valid credentials. Defenders mitigate this by enforcing server-side session validation and never trusting client-controlled inputs (cookies, headers, URL parameters) as proof of authentication state.

## Key facts
- CWE-287 is the parent of many specific authentication failures, including CWE-306 (missing authentication) and CWE-290 (authentication bypass via spoofing)
- Common patterns: checking only a username without verifying the password hash, accepting a "logged_in=true" cookie without server-side session confirmation, or skipping MFA for certain code paths
- Frequently exploited via forced browsing, parameter tampering, or JWT "alg:none" attacks where the signature verification is bypassed entirely
- Maps to OWASP Top 10 **A07:2021 – Identification and Authentication Failures**
- CVSS scores for improper authentication vulnerabilities commonly reach **Critical (9.0+)** when unauthenticated remote access to privileged functions is possible
