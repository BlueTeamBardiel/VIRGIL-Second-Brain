# CWE-1258

## What it is
Like a safe with a glass door — the lock works perfectly, but anyone walking by can see exactly what's inside. CWE-1258 describes the exposure of sensitive system state or debug information through registers, memory, or hardware interfaces that remain readable even when they shouldn't be. This occurs when hardware or firmware fails to properly restrict access to internal state data during normal or privileged operation transitions.

## Why it matters
An attacker with physical or low-level software access to a device (e.g., via JTAG or debug ports left enabled in production firmware) can read CPU registers or memory-mapped I/O that exposes cryptographic keys, authentication tokens, or privilege levels mid-operation. This was a real concern in embedded IoT devices where manufacturers shipped hardware with debug interfaces active, allowing researchers to extract firmware secrets without decapping the chip.

## Key facts
- Commonly manifests in **hardware debug interfaces** (JTAG, UART, SWD) left enabled in production builds, exposing internal register states
- Violates the principle of **least privilege at the hardware level** — sensitive state should only be readable by authorized entities and contexts
- Distinct from software information disclosure — this weakness lives at the **firmware/hardware boundary**
- Can leak **cryptographic key material** held in registers during encryption operations if an attacker can pause or inspect execution state
- Mitigated by **disabling or locking debug interfaces** via fuse bits or secure boot enforcement before production deployment
