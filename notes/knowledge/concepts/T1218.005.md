# T1218.005

## What it is
Like using a building's own security guard to smuggle contraband past the front desk, `mshta.exe` is a legitimate Windows binary that attackers weaponize to execute malicious HTML Applications (HTA files) while bypassing security controls. Precisely, this sub-technique involves abusing the Microsoft HTML Application Host (`mshta.exe`) — a signed, trusted Windows binary — to run `.hta` files or inline VBScript/JScript, effectively living off the land.

## Why it matters
In the 2017 FIN7 campaigns targeting financial institutions, attackers delivered phishing emails containing `.hta` attachments; when opened, `mshta.exe` executed embedded VBScript that downloaded and ran Carbanak malware. Because `mshta.exe` is a legitimate signed Microsoft binary, many older AV solutions whitelisted it entirely, allowing the payload to run undetected on hundreds of bank endpoints.

## Key facts
- `mshta.exe` resides at `C:\Windows\System32\mshta.exe` and is signed by Microsoft, making it a classic **LOLBin** (Living Off the Land Binary)
- Can execute code directly from the command line: `mshta.exe vbscript:Execute("malicious code")(window.close)` — no file drop required
- HTA files run with **Internet Explorer's engine** but execute with full user-level OS permissions, bypassing browser sandboxes
- Defenders can block execution via **AppLocker or WDAC policies** targeting `mshta.exe`, or monitor for parent process anomalies (e.g., `mshta.exe` spawned by `winword.exe`)
- Associated with threat groups: **FIN7, APT32, Lazarus Group** — making it a high-priority detection target in enterprise environments
