# CWE-307

## What it is
Like a bank vault that lets you try every combination on the dial with no lockout or alarm, CWE-307 is the failure to implement adequate restrictions on repeated authentication attempts. It allows an attacker to systematically guess credentials without the system pushing back.

## Why it matters
In the 2012 LinkedIn breach aftermath, many exposed password hashes were cracked offline — but CWE-307 enables *online* attacks where an attacker repeatedly submits guesses directly against a live login endpoint. A web application without account lockout, rate limiting, or CAPTCHA allows automated tools like Hydra or Burp Intruder to run thousands of credential attempts per minute against real user accounts.

## Key facts
- **Brute force and credential stuffing** are the primary attack vectors enabled by CWE-307; without restrictions, automation trivializes authentication
- NIST SP 800-63B recommends **rate limiting** and **lockout policies** (e.g., no more than 100 failed attempts before throttling) rather than hard lockouts, which can enable denial-of-service
- A hard lockout after 3-5 attempts can itself become a **DoS vector** against legitimate users — intelligent throttling (exponential backoff) is preferred
- CWE-307 is distinct from CWE-308 (Use of Single-Factor Authentication) and CWE-309 (Use of Password Systems); it specifically targets the *absence of attempt limiting*
- Mitigations include: CAPTCHA challenges, per-IP rate limiting, account lockout with unlock workflows, and **multi-factor authentication** as a compensating control
