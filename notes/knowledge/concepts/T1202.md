# T1202

## What it is
Like a smuggler hiding contraband inside a legitimate shipping container, attackers use trusted system utilities to execute malicious commands indirectly. T1202 (Indirect Command Execution) refers to leveraging built-in OS tools — such as `forfiles`, `pcalua.exe`, or `bash.exe` on Windows — to run commands without directly invoking `cmd.exe` or `powershell.exe`, bypassing controls that monitor those obvious entry points.

## Why it matters
During a red team engagement, an attacker blocked by PowerShell Constrained Language Mode used `forfiles /c "cmd /c calc.exe"` to execute code without touching PowerShell directly, evading endpoint detection tuned to watch for standard shell launches. Defenders who only monitor `cmd.exe` spawning chains miss this entirely, making process tree analysis critical for detection.

## Key facts
- **Common LOLBins used:** `forfiles.exe`, `pcalua.exe` (Program Compatibility Assistant), `bash.exe` (WSL), `msiexec.exe`, and `syncappvpublishingserver.exe`
- **Detection gap:** Many SIEMs alert on `cmd.exe` or `powershell.exe` as parent processes but miss these indirect launchers — attackers exploit this tuning blind spot
- **OS support:** This technique is Windows-native but conceptually applies across platforms wherever scripting engines or compatibility shims exist
- **MITRE sub-technique context:** Sits under the **Defense Evasion** tactic (TA0005), specifically designed to circumvent command-line monitoring and application whitelisting
- **Key detection strategy:** Monitor for unusual parent-child process relationships using Sysmon Event ID 1 (Process Create) and correlate with command-line argument logging enabled via Group Policy
