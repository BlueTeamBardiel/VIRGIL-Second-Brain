# T1078.001

## What it is
Like a skeleton key that was forged before the building was even built, default credentials are the factory-set usernames and passwords baked into hardware and software at manufacturing time. Specifically, T1078.001 describes adversaries using these unchanged default accounts — think `admin/admin` or `cisco/cisco` — to gain valid, legitimate-looking access to systems without ever cracking a single password.

## Why it matters
The Mirai botnet (2016) weaponized this technique at massive scale, scanning the internet for IoT devices still running factory defaults and conscripting them into a botnet that launched the largest DDoS attack recorded at the time, taking down Dyn DNS and disrupting Twitter, Reddit, and Netflix. Defenders counter this by requiring credential changes on first boot and scanning their own networks with tools like Shodan or Nessus to identify exposed default-credential devices before attackers do.

## Key facts
- Mitre classifies this under the broader **Valid Accounts (T1078)** technique, specifically targeting the "Default Accounts" sub-category
- Default credentials are catalogued publicly at sites like **DefaultCreds-cheat-sheet** and **CIRT.net** — attackers use these as lookup tables
- This technique produces **minimal anomalous log noise** because authentication succeeds — defenders must look for logins from unexpected IPs or at odd hours
- Affected assets include routers, switches, IP cameras, printers, databases (MySQL default: `root/[blank]`), and web app consoles (Tomcat: `tomcat/tomcat`)
- CIS Control 5 (Account Management) directly mitigates this by mandating default password changes as a foundational hygiene requirement
