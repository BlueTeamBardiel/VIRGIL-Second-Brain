# T1021.008

## What it is
Like using a building's internal intercom system to move between floors without ever stepping outside, Direct Cloud VM Connections allow attackers to pivot laterally within cloud environments using native management protocols (SSH, RDP) directly to virtual machine instances — bypassing traditional network perimeters entirely. This technique (MITRE ATT&CK T1021.008) exploits cloud-native tools like AWS Systems Manager Session Manager, Azure Bastion, or GCP's IAP tunneling to establish interactive sessions with VMs using IAM credentials rather than exposed network ports.

## Why it matters
In the 2023 SCARLETEEL campaign, threat actors who compromised cloud credentials used AWS SSM Session Manager to tunnel into EC2 instances that had *no public IP address and no open security group ports* — making traditional network-based detection completely blind to the lateral movement. Defenders who focused only on firewall logs missed the intrusion entirely; detection required auditing CloudTrail logs for `ssm:StartSession` API calls.

## Key facts
- Attackers need valid IAM credentials with `ssm:StartSession` or equivalent permissions — credential theft is the prerequisite
- No inbound ports need to be open on the target VM; traffic flows outbound through the cloud provider's control plane
- Detection relies on cloud audit logs (CloudTrail, Azure Monitor, GCP Audit Logs) — NOT network flow data or firewall logs
- Legitimate use is common (cloud admins use these tools daily), making behavioral baselining critical to reduce false positives
- Mitigations include least-privilege IAM policies, restricting Session Manager via Service Control Policies (SCPs), and requiring MFA for sensitive API calls
