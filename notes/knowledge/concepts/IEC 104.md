# IEC 104

## What it is
Think of it as SCADA's postal service that moved from private couriers (serial lines) to the public highway (TCP/IP) — suddenly everyone can reach the sorting office. IEC 104 (formally IEC 60870-5-104) is a telecontrol protocol that adapts IEC 101 for transmission over TCP/IP networks (port 2404), enabling remote monitoring and control of electrical grid substations and industrial equipment. It carries Application Protocol Data Units (APDUs) containing commands and data objects to control circuit breakers, transformers, and RTUs in real time.

## Why it matters
In the 2015 Ukraine power grid attack, adversaries used the BlackEnergy malware combined with a modified KillDisk payload — but critically, operators observed attackers issuing IEC 104 commands directly to substation RTUs to open breakers and cut power to ~225,000 customers. Because IEC 104 carries no native authentication, a threat actor who achieves network access to an OT segment can send legitimate-looking control commands indistinguishable from real operator traffic.

## Key facts
- **Port 2404/TCP** is the standard listening port for IEC 104 servers (RTUs/IEDs)
- **No built-in authentication or encryption** in the base standard — any host on the network can issue control commands
- Uses a **connection-oriented, session-based model** with U-frames (control), S-frames (supervisory), and I-frames (information/data)
- **Common Object Address (COA)** and **Information Object Address (IOA)** uniquely identify data points within a station hierarchy
- IEC 62351 was developed specifically to add TLS-based security to IEC 104, but adoption in legacy OT environments remains low
