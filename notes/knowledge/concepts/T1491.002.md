# T1491.002

## What it is
Like a graffiti artist defacing a storefront's sign to humiliate the owner publicly, attackers replace or alter an organization's external-facing website content to send a political, ideological, or taunting message. Defacement (External) is the deliberate modification of public-facing web content — typically homepages — to display attacker-controlled messaging, logos, or propaganda.

## Why it matters
During the 2022 Russian-Ukrainian conflict, multiple Ukrainian government websites were defaced with threatening messages before cyberattacks escalated, serving as both psychological warfare and a distraction from simultaneous destructive operations. Defenders must monitor for unauthorized file changes on web servers using integrity monitoring tools (e.g., Tripwire) to catch defacement before it erodes public trust.

## Key facts
- Classified under **Impact** (TA0040) in the MITRE ATT&CK framework — the goal is disruption, reputation damage, or propaganda, not data theft
- Attackers typically gain initial access via web shell upload, CMS vulnerabilities (WordPress, Joomla), or stolen credentials before modifying index files
- Defacement is often performed by **hacktivists** (e.g., Anonymous-affiliated groups) but also nation-state actors using it as a psychological operations tool
- Key detection artifacts: unexpected modification timestamps on HTML/PHP files, new files in web root directories, and web server access logs showing unusual POST requests
- Differs from **T1491.001 (Internal Defacement)**, which targets internal systems like intranet pages or digital signage to demoralize employees rather than the public
