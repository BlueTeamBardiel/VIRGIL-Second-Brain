# T1046

## What it is
Like a burglar walking down a street testing every door handle before deciding which house to rob, network service scanning is the systematic probing of IP addresses and ports to discover what services are running on a target network. Precisely defined, T1046 (Network Service Discovery) involves adversaries using tools like Nmap, Masscan, or built-in OS commands to enumerate open ports, running services, and version information on reachable hosts.

## Why it matters
During the SolarWinds supply chain attack, threat actors performed internal network discovery after initial compromise to map high-value targets like Active Directory servers and security tools before moving laterally. Defenders counter this by monitoring for anomalous scanning patterns — sudden spikes in connection attempts across sequential ports from a single internal host are a reliable detection signal in a SIEM.

## Key facts
- Common tools used: **Nmap** (comprehensive), **Masscan** (high-speed), **Netcat**, and native commands like `net view` or `ss -tulnp`
- Scanning techniques include SYN (half-open) scans, UDP scans, and version detection (`-sV` in Nmap) — each with different stealth profiles
- T1046 is categorized under the **Discovery** tactic in the MITRE ATT&CK framework
- Detection signatures include high volumes of RST/ACK responses, sequential port connections, and SMB enumeration traffic (port 445)
- Defenders can deploy **honeypots** and **deception ports** — any connection to a non-advertised port is an immediate IOC
