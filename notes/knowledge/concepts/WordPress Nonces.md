# WordPress Nonces

## What it is
Like a bouncer stamping your hand at the door — the stamp is only valid for that night, and you can't fake it to get back in tomorrow. A WordPress nonce ("number used once") is a time-limited, action-specific token embedded in URLs and forms to verify that a submitted request was intentionally generated by a legitimate, authenticated user session.

## Why it matters
Without nonces, a WordPress admin could be tricked into clicking a malicious link that silently creates a new admin account — a classic Cross-Site Request Forgery (CSRF) attack. Because the nonce ties the request to a specific user, action, and time window, the forged request from an external site fails validation even if the victim is fully authenticated. This is why WordPress nonce verification is a required security control for any plugin that modifies data.

## Key facts
- WordPress nonces are **not** truly "used once" — they are valid for **24 hours** by default (two 12-hour cycles), which is a common misconception and exam trap
- Nonces are generated using `wp_create_nonce('action-name')` and verified with `check_ajax_referer()` or `wp_verify_nonce()` — missing verification is a top plugin vulnerability class
- They are tied to the **logged-in user ID, action string, and time period** — all three must match for validation to succeed
- Nonces do **not** protect against replay attacks within the validity window, which is why they are a CSRF mitigation, not an authentication mechanism
- Unvalidated nonces in AJAX handlers are one of the most commonly reported WordPress CVEs, frequently leading to **privilege escalation** by unauthenticated users
