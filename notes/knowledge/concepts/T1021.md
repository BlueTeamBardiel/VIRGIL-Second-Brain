# T1021

## What it is
Like using a spare key hidden under the doormat instead of breaking a window — attackers use legitimate remote access services that are already built into the operating system to move between machines without triggering alarms. T1021 (Remote Services) covers the abuse of protocols like RDP, SSH, SMB, VNC, and WinRM to laterally move through a network using valid credentials.

## Why it matters
During the 2021 Colonial Pipeline ransomware attack, attackers leveraged a compromised VPN account to pivot internally — a textbook T1021 scenario where legitimate remote access became the attacker's highway. Defenders monitoring for T1021 focus on unusual authentication times, geographic anomalies, and service accounts authenticating interactively, because the traffic itself looks perfectly normal.

## Key facts
- **Sub-techniques include:** RDP (T1021.001), SMB/Windows Admin Shares (T1021.002), Distributed Component Object Model (T1021.003), SSH (T1021.004), VNC (T1021.005), and Windows Remote Management (T1021.006)
- **Credential dependency:** T1021 almost always requires prior credential access — stolen hashes, cleartext passwords, or Kerberos tickets
- **RDP (port 3389)** is the most commonly abused sub-technique and a major target for brute-force and pass-the-hash attacks
- **Detection pivot:** Look for EventID 4624 (Type 3 or Type 10 logons) paired with lateral movement indicators across multiple hosts
- **Mitigation:** Network segmentation, disabling unnecessary remote services, MFA on RDP/VPN, and privileged access workstations (PAWs) all reduce T1021 exposure
