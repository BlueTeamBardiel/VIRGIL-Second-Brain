# CWE-434

## What it is
Like a nightclub bouncer who checks IDs but never actually looks at the person's face — only the name on the card — a server that validates a filename's extension without inspecting the file's actual content can be fooled completely. CWE-434 is the **Unrestricted Upload of File with Dangerous Type**: a vulnerability where an application accepts file uploads without adequately verifying that the file's type, content, or extension cannot lead to malicious execution.

## Why it matters
In the 2017 compromise of numerous WordPress sites, attackers exploited plugins with unrestricted upload flaws to upload PHP webshells disguised with double extensions (e.g., `shell.php.jpg`). Once uploaded to a web-accessible directory, the attacker sent an HTTP request directly to the file path, triggering server-side code execution and achieving full remote access — all starting with a single file upload form.

## Key facts
- **Magic bytes matter**: Attackers rename malicious files (e.g., `.php` backdoors) with trusted extensions like `.jpg`; defenses must validate MIME type via file content (magic bytes), not just extension.
- **Double extensions** (e.g., `evil.php.jpg`) exploit misconfigured web servers that process the first recognized extension.
- **Upload directory permissions** are a critical control: storing uploads *outside* the web root or disabling script execution in upload folders neutralizes many exploits.
- **CVSS scores for this CWE commonly reach 9.8 (Critical)** when the upload directory is web-accessible and the server executes the file type.
- Mitigation requires a **whitelist approach** (allow only specific safe MIME types and extensions) rather than a blacklist, which attackers routinely bypass.
