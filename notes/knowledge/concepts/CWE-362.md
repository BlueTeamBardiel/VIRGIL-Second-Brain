# CWE-362

## What it is
Imagine two chefs simultaneously reaching into the same pantry and both grabbing the last egg — each assuming the other hasn't touched it yet. CWE-362 (Race Condition / Concurrent Execution Using Shared Resource with Improper Synchronization) occurs when a program's behavior depends on the sequence or timing of uncontrollable events, allowing an attacker to exploit the window between a check and its corresponding action (TOCTOU — Time of Check to Time of Use).

## Why it matters
The classic attack involves a privileged program checking whether a file is safe to open, then an attacker swapping that file with a symlink to `/etc/passwd` before the program actually opens it. This TOCTOU window has been exploited in real Unix setuid programs to escalate privileges — the check says "safe," but by use-time, the resource has been swapped for something malicious.

## Key facts
- **TOCTOU** is the most common manifestation: the gap between checking a resource and using it is the attack surface
- Race conditions are particularly dangerous in **setuid/setgid programs** because the vulnerable process runs with elevated privileges
- Multithreaded applications without proper **mutex locks or semaphores** are prime targets for race condition exploitation
- CWE-362 is a parent of several child weaknesses including **CWE-367** (TOCTOU Race Condition) and **CWE-364** (Signal Handler Race Condition)
- Mitigations include **atomic operations**, using file descriptors instead of filenames after initial check, and applying the principle of least privilege to reduce blast radius
