# CWE-200

## What it is
Like a bank teller accidentally reading your account balance aloud in a crowded lobby, this weakness occurs when a system exposes information to actors who aren't authorized to receive it. CWE-200 (Exposure of Sensitive Information to an Unauthorized Actor) is a broad category covering any unintended disclosure of data — through error messages, logs, API responses, or improper access controls.

## Why it matters
A classic example is verbose error messages in web applications: when a login form returns "incorrect password" vs. "user not found," attackers can enumerate valid usernames and focus brute-force attacks. Developers often consider detailed errors helpful for debugging, but in production these messages hand adversaries a reconnaissance roadmap. Proper error handling that returns generic messages ("invalid credentials") closes this channel.

## Key facts
- CWE-200 is a **parent category** in MITRE's CWE hierarchy; child weaknesses include CWE-209 (error message info leakage), CWE-312 (cleartext storage), and CWE-319 (cleartext transmission)
- Common sources: HTTP response headers (server version, framework), stack traces, debug logs, directory listings, and overly verbose API errors
- Classified under **OWASP Top 10 A02:2021 – Cryptographic Failures** and contributes to **A05:2021 – Security Misconfiguration**
- On Security+/CySA+ exams, this maps to **information disclosure** threats and violations of the **confidentiality** pillar of the CIA triad
- Mitigations include least-privilege data access, suppressing production debug output, sanitizing log files, and implementing proper data classification policies
