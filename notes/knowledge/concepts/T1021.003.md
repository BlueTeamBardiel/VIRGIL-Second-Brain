# T1021.003

## What it is
Like using a building's own intercom system to unlock doors from the inside, DCOM allows attackers to execute code on remote machines by abusing Windows' built-in distributed application framework. Distributed Component Object Model (DCOM) is a Microsoft protocol that enables software components to communicate across networked computers, and adversaries exploit it to move laterally without dropping traditional malware payloads.

## Why it matters
During the NotPetya outbreak, lateral movement techniques leveraging Windows interoperability protocols — including DCOM — allowed the worm to spread rapidly across corporate networks without needing stolen credentials in every case. Defenders who blocked SMB (port 445) sometimes missed DCOM traffic on port 135 (RPC endpoint mapper), giving attackers a secondary pivot path that bypassed perimeter controls.

## Key facts
- DCOM uses **TCP port 135** as the RPC endpoint mapper, with dynamic high ports assigned for actual data transfer — making firewall rules tricky
- Attackers commonly abuse the **MMC20.Application** and **ShellWindows** COM objects to spawn remote processes without touching disk
- Requires **local administrator privileges** on the target machine, meaning credential theft (e.g., Pass-the-Hash) usually precedes DCOM lateral movement
- Unlike PsExec, DCOM abuse leaves **fewer traditional IOCs** — no new services created, no dropped binaries
- Detectable via **Windows Event ID 4624** (logon) combined with unusual DCOM-related process spawning in Sysmon logs (Event ID 10/3)
