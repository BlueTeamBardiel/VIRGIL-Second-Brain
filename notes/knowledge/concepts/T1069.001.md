# T1069.001

## What it is
Like a new employee quietly reading the org chart to figure out who has the keys to the server room, adversaries enumerate local groups to map out privilege boundaries on a compromised host. Specifically, T1069.001 covers discovery of local groups and their memberships — particularly high-value groups like `Administrators` — on a single system using built-in OS commands.

## Why it matters
During the 2020 SolarWinds intrusion, threat actors performed systematic local group enumeration post-compromise to identify which accounts held administrative rights before moving laterally. Defenders monitoring for sudden bursts of `net localgroup administrators` or `Get-LocalGroupMember` commands in process logs can catch this reconnaissance phase before privilege escalation occurs.

## Key facts
- **Primary commands used:** `net localgroup`, `net localgroup administrators`, `whoami /groups`, and PowerShell's `Get-LocalGroupMember` on Windows; `groups`, `id`, and `cat /etc/group` on Linux
- **MITRE tactic:** Discovery (TA0007) — this is reconnaissance *after* initial access, not before
- **Defender detection:** Alert on `net.exe` or `net1.exe` spawning from unusual parent processes (e.g., Word, cmd spawned by a web service)
- **Why Administrators group specifically:** Membership reveals which accounts can be leveraged for privilege escalation or lateral movement without needing a new exploit
- **Logging requirement:** Windows Security Event ID **4798** (local group membership enumerated) and **4799** are the key telemetry sources — disabled by default in many environments
