# T1068

## What it is
Like a janitor who finds a broken lock on the executive suite and lets himself in, exploitation for privilege escalation abuses a software vulnerability to jump from a low-privilege account to a higher one — often SYSTEM or root. Precisely: T1068 covers attackers leveraging bugs in OS kernels, drivers, or services to gain elevated permissions they were never authorized to hold.

## Why it matters
In the 2021 PrintNightmare (CVE-2021-34527) attacks, threat actors exploited a Windows Print Spooler vulnerability to escalate from a standard domain user to SYSTEM, then deployed ransomware and moved laterally across entire networks. Defenders who had patched promptly or disabled the Print Spooler service on non-print servers were completely unaffected — a textbook case where patch cadence directly determined blast radius.

## Key facts
- **Common targets**: Windows kernel drivers, misconfigured SUID binaries on Linux, unpatched service processes running as SYSTEM/root
- **Dirty Pipe (CVE-2022-0847)** is a notable Linux kernel privilege escalation — any local user could overwrite read-only files and gain root
- **Detection signal**: Process lineage anomalies — a web server process spawning a shell running as SYSTEM is a high-fidelity indicator
- **Mitigation**: Patch management, Exploit Protection (Windows Defender), mandatory access controls (SELinux/AppArmor), and least-privilege service accounts all reduce attack surface
- **Distinguishes from T1548**: T1068 *exploits a vulnerability*; T1548 (Abuse Elevation Control Mechanism) abuses *legitimate OS features* like sudo or UAC bypass
