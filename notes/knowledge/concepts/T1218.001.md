# T1218.001

## What it is
Like using a museum's own security guard to smuggle out a painting, attackers leverage **Compiled HTML Help (CHH) files** — legitimate Windows `.chm` files — to execute malicious code through a trusted Microsoft binary. Specifically, adversaries abuse `hh.exe`, the Windows HTML Help executable, to run embedded scripts or download payloads while bypassing application whitelisting controls.

## Why it matters
In 2017, APT32 (OceanLotus) distributed malicious `.chm` files via spearphishing emails targeting Southeast Asian organizations. When victims opened the file, `hh.exe` silently executed embedded JavaScript that downloaded a backdoor — the entire chain blessed by Windows as "legitimate" because `hh.exe` is a signed Microsoft binary.

## Key facts
- `hh.exe` is the signed Microsoft binary that processes `.chm` files; it can execute `<OBJECT>` tags containing ActiveX or scripting content
- Attackers embed VBScript or JavaScript inside `.chm` files using the `HHCTRL.OCX` ActiveX control to achieve code execution
- This technique bypasses **AppLocker** and **Software Restriction Policies** because `hh.exe` is whitelisted by default
- Malicious `.chm` files are frequently delivered via phishing emails, exploiting user trust in "help documentation"
- Detection focus: monitor for `hh.exe` spawning unexpected child processes (e.g., `cmd.exe`, `powershell.exe`, `wscript.exe`) or making outbound network connections
