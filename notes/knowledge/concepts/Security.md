# Security

## What it is
Like a medieval castle with walls, a moat, guards, and a drawbridge — security is never one thing but a layered system of controls. Precisely: security is the practice of protecting information systems, assets, and data from unauthorized access, disclosure, alteration, or destruction through technical, administrative, and physical controls. The goal is preserving the CIA triad — Confidentiality, Integrity, and Availability.

## Why it matters
In 2017, the Equifax breach exposed 147 million Americans' personal data because an unpatched Apache Struts vulnerability went unaddressed for months — a failure of patch management, network segmentation, and monitoring simultaneously. This illustrates that security is a *system*: one broken layer cascades into catastrophic compromise. A properly layered defense (patching + IDS + least privilege) would have stopped or contained the attacker at multiple points.

## Key facts
- The **CIA Triad** is the foundational model: Confidentiality (only authorized access), Integrity (data unchanged without authorization), Availability (systems accessible when needed)
- Security operates across three control categories: **Technical** (firewalls, encryption), **Administrative** (policies, training), and **Physical** (locks, badges)
- Controls are further classified by function: **Preventive, Detective, Corrective, Deterrent, Compensating, and Directive**
- **Defense in Depth** is the principle of layering multiple independent controls so no single failure creates total compromise
- The **principle of least privilege** states that users and systems should have only the minimum access required to perform their function — reducing attack surface
