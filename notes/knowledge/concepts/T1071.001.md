# T1071.001

## What it is
Think of a spy sending secret messages hidden inside ordinary postcards — the postal service never suspects because postcards are normal mail. Application Layer Protocol: Web Protocols (T1071.001) is exactly this: attackers tunnel C2 communications through HTTP or HTTPS traffic, blending malicious data into the most common protocol on any network so it looks like normal web browsing.

## Why it matters
During the APT41 campaigns, operators used HTTPS-based C2 channels to exfiltrate data from compromised networks — security teams watching for suspicious ports saw nothing unusual because port 443 traffic was expected and largely trusted. Defenders must inspect *what's inside* HTTPS, not just *that* HTTPS exists, which requires SSL/TLS inspection proxies and behavioral anomaly detection rather than simple port-based firewall rules.

## Key facts
- HTTP/HTTPS are used because they're almost universally allowed outbound through firewalls, making blocking impractical without breaking legitimate business functions
- Malware families like Cobalt Strike, Emotet, and SUNBURST (SolarWinds) all used HTTP/HTTPS for C2 communication
- Detection relies on anomaly signals: beaconing patterns (regular check-in intervals), unusual User-Agent strings, connections to newly registered domains, and abnormal data volumes
- HTTPS makes this particularly dangerous — without TLS inspection, blue teams are blind to payload content
- MITRE classifies this under the **Command and Control** tactic (TA0011); parent technique is T1071 (Application Layer Protocol)
