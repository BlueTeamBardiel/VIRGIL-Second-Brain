# T1059.005

## What it is
Like a contractor using a building's own master key to bypass security rather than breaking a window, Visual Basic Script lets attackers use a tool already trusted by Windows to execute malicious code. VBScript (T1059.005) is a scripting language natively supported by Windows via the Windows Script Host (wscript.exe/cscript.exe), enabling attackers to run scripts for automation, persistence, and payload delivery without installing foreign tools.

## Why it matters
The Emotet banking trojan famously used malicious VBScript embedded in Office macro-enabled documents to download and execute second-stage payloads. When a user enabled macros, the VBScript contacted a C2 server, retrieved the Emotet loader, and established persistence — all using Windows-native components that many older security tools whitelisted by default.

## Key facts
- VBScript runs through **wscript.exe** (GUI, no console window) or **cscript.exe** (command-line), making wscript.exe a high-value process to monitor
- Microsoft began **disabling VBScript by default in Internet Explorer and Office** starting in 2023–2024, significantly reducing the attack surface but leaving legacy systems exposed
- Common delivery vectors: malicious email attachments (.vbs files), Office macros, and HTML Application (.hta) files
- VBScript can invoke **WMI, COM objects, and the Windows Shell** to spawn child processes, download files, and modify the registry
- Detection focus: monitor for **wscript.exe or cscript.exe spawning child processes** like PowerShell, cmd.exe, or network-calling utilities — this is a strong behavioral indicator
