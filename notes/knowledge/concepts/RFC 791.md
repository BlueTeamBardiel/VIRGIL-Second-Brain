# RFC 791

## What it is
Think of IP like the postal system's sorting rules: it defines the envelope format, addressing scheme, and delivery instructions — but makes zero promises the letter arrives. RFC 791 is the 1981 specification defining the Internet Protocol version 4 (IPv4), establishing how packets are addressed, fragmented, and routed across networks without connection state.

## Why it matters
The fragmentation mechanism defined in RFC 791 became a notorious attack surface — the **Teardrop attack** exploited overlapping fragment offsets to crash target systems by sending malformed IP fragments the OS couldn't reassemble. Defenders use IDS rules and firewall fragment reassembly to detect and block malicious fragmentation patterns before packets reach vulnerable hosts.

## Key facts
- IPv4 addresses are **32 bits** (4 octets), yielding ~4.3 billion possible addresses — exhaustion drove adoption of NAT and IPv6
- The IP header contains a **TTL (Time to Live)** field: each router decrements it by 1; at 0, the packet is dropped (prevents infinite routing loops)
- **Fragmentation** is controlled by three header fields: *Flags* (DF = Don't Fragment, MF = More Fragments), *Fragment Offset*, and *Identification* — all exploitable for evasion and DoS
- The header **checksum** covers only the header, not the payload — data integrity is the transport layer's problem (TCP/UDP)
- IP provides **no authentication** of source addresses, enabling **IP spoofing** — a foundational technique in DDoS amplification and blind injection attacks
