# Posture Assessment

## What it is
Like a doctor reviewing your full medical history, vitals, and lifestyle habits before diagnosing illness, a posture assessment examines an organization's entire security state before attackers find the weaknesses first. It is a comprehensive evaluation of an organization's current security controls, configurations, policies, and practices to identify gaps between its actual security state and its desired or required security baseline.

## Why it matters
Before the 2013 Target breach, an assessment of their vendor access controls and network segmentation would have revealed that third-party HVAC credentials could pivot into the cardholder data environment — a critical posture gap that cost $162 million. Regular posture assessments help organizations prioritize remediation efforts before adversaries exploit misconfigurations, unpatched systems, or excessive privileges.

## Key facts
- Posture assessments differ from penetration testing — they measure *breadth* of risk across the environment, not depth of a single exploitable path
- Outputs typically include a risk register, control gap analysis, and a prioritized remediation roadmap aligned to frameworks like NIST CSF or CIS Controls
- Security Information and Event Management (SIEM) data, vulnerability scan results, and configuration audits are all common inputs to a posture assessment
- CySA+ expects candidates to distinguish posture assessments from point-in-time audits — posture is ongoing and continuous, not a one-time snapshot
- Cloud Security Posture Management (CSPM) tools automate posture assessment for cloud environments, catching misconfigurations like publicly exposed S3 buckets in real time
