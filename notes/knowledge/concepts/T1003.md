# T1003

## What it is
Like a thief who doesn't pick a lock but instead steals the locksmith's master key mold, OS Credential Dumping extracts authentication credentials directly from operating system memory or storage rather than guessing or cracking them. It is the MITRE ATT&CK technique describing adversary attempts to obtain account login credentials — hashes, plaintext passwords, or Kerberos tickets — from the OS itself.

## Why it matters
In the 2020 SolarWinds supply chain attack, threat actors used credential dumping via tools like Mimikatz to harvest NTLM hashes and Kerberos tickets from compromised hosts, enabling lateral movement across federal agency networks without triggering traditional brute-force alerts. Defenders who monitored LSASS memory access patterns could detect the technique even when malware signatures were unknown.

## Key facts
- **LSASS (Local Security Authority Subsystem Service)** is the primary target on Windows — it holds credential material in memory and is accessed by tools like Mimikatz (`sekurlsa::logonpasswords`)
- Sub-techniques include: LSASS Memory (T1003.001), SAM Database (T1003.002), NTDS (T1003.003), LSA Secrets (T1003.004), and /etc/passwd + /etc/shadow on Linux (T1003.008)
- **Pass-the-Hash** attacks are enabled directly by credential dumping — stolen NTLM hashes can authenticate without cracking
- Windows Credential Guard (virtualization-based security) specifically mitigates LSASS dumping by isolating credentials in a protected enclave
- Detection methods include monitoring for suspicious LSASS process access (`OpenProcess` calls), Sysmon Event ID 10, and unexpected `procdump.exe` or `comsvcs.dll` usage
