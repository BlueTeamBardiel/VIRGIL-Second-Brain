# SHA-256

## What it is
Like a meat grinder that turns any steak into identical-sized pellets — you can't reverse the pellets back into steak, and even swapping one gram of meat produces completely different pellets. SHA-256 (Secure Hash Algorithm 256-bit) is a one-way cryptographic hash function that takes arbitrary-length input and produces a fixed 256-bit (32-byte) digest, making it computationally infeasible to reverse or find two inputs producing the same output.

## Why it matters
In 2017, the "SHAttered" attack demonstrated SHA-1 collision feasibility, forcing the industry to migrate toward SHA-256. Today, certificate authorities exclusively issue TLS certificates using SHA-256 signatures — if an attacker could forge a collision, they could present a malicious certificate that browsers accept as legitimate, enabling undetected MITM attacks against HTTPS traffic.

## Key facts
- Produces a **256-bit (64 hexadecimal character)** digest regardless of input size — hashing "a" or the entire Library of Congress yields the same output length
- Member of the **SHA-2 family**, designed by the NSA and standardized by NIST in FIPS 180-4
- Exhibits the **avalanche effect**: changing even one input bit flips approximately 50% of the output bits
- Used in **Bitcoin's proof-of-work**, **TLS certificate signatures**, **file integrity verification**, and **password storage** (typically combined with salting via bcrypt or PBKDF2)
- **Not a password storage solution alone** — raw SHA-256 is fast by design, making it vulnerable to GPU-accelerated brute-force attacks; dedicated KDFs should be used instead
- Collision resistance is considered **computationally secure** (no practical collision attack exists as of 2024), unlike its predecessor SHA-1
