# 2.2 - Common Threat Vectors

Understanding threat vectors is fundamental to CompTIA Security+ and real-world cybersecurity defense. A threat vector (or attack vector) is any method an attacker uses to gain unauthorized access to or infect a target system—and there are far more of them than most people realize. This section covers the major categories: message-based, file-based, image-based, voice-based, and removable device vectors. Mastering these vectors is critical because they form the foundation of your defensive strategy; you cannot protect what you don't understand, and the exam heavily tests your ability to identify which vector a given attack scenario exploits.

---

## Key Concepts

- **Threat Vector (Attack Vector)**: A specific method or pathway an attacker uses to exploit a vulnerability and gain access to a system or deliver [[Malware]]. Also called an "attack surface."

- **Message-Based Vectors**: Attacks delivered through electronic messaging (email, SMS, IM):
  - [[Phishing]]: Social engineering attacks using deceptive links or attachments sent via email, text, or instant message
  - **Social Engineering**: Manipulation tactics like invoice scams, cryptocurrency scams, and pretexting
  - **Attachment Risk**: Malicious payloads embedded in emails—scanning and validation are critical defenses

- **File-Based Vectors**: Malicious code hidden within seemingly legitimate files:
  - **PDF Files**: Portable Document Format can contain embedded objects (JavaScript, forms) used to deliver exploits
  - **Microsoft Office Documents**: Word, Excel, and PowerPoint files with embedded macros or add-ins that execute code
  - **Compression Archives** (ZIP, RAR, 7z): Container files that hide multiple payloads
  - **Executable Files**: Traditional .exe, .dll, .sys files—the most obvious but still dangerous

- **Image-Based Vectors**: Threats embedded in or delivered via image files:
  - **SVG (Scalable Vector Graphics)**: XML-based format that can contain malicious scripts (HTML injection, [[JavaScript]] attacks)
  - **HTML Injection**: Embedding malicious HTML or JavaScript within image metadata or SVG code
  - **Input Validation Gap**: Browsers must sanitize image content; improper validation allows code execution
  - **Steganography**: Hidden payloads within legitimate image data (not always detected by traditional scanning)

- **Voice Call Vectors**: Attacks delivered over phone/VoIP infrastructure:
  - **Vishing**: Voice [[Phishing]]—social engineering over the phone to extract credentials or sensitive information
  - **Spam over IP (SPAM over IP)**: Large-scale automated malicious phone calls
  - **War Dialing**: Automated scanning of phone number ranges to find modems or vulnerable systems (legacy but still active)
  - **Call Tampering**: Disrupting or intercepting voice calls (integrity/confidentiality threat)

- **Removable Device Vectors**: Physical devices that bypass network perimeter controls:
  - **USB Flash Drives**: Can contain [[Malware]], bypass [[Firewall]] protections, and infect air-gapped or high-security networks
  - **USB Keyboard Injection (Hacker on a Chip)**: USB devices programmed to act as input devices, automatically executing commands
  - **Data Exfiltration**: Malicious insiders or compromised hosts copying terabytes of data to USB with zero bandwidth detection
  - **Air-Gapped Systems**: Industrial control systems, classified networks, and critical infrastructure vulnerable to physical removable media

## How It Works (Feynman Analogy)

Imagine a bank with multiple doors and windows: the front door (email), the loading dock (removable devices), the phone line (voice calls), and even the decorative posters on the wall (images). A burglar doesn't need to smash through the main entrance—they can slip through the loading dock, call ahead pretending to be a delivery driver, or use a disguised entry point you didn't think to guard.

**Technically**: An attacker chooses the path of least resistance. A user may have been trained not to click suspicious links in email, so the attacker embeds malicious code in a PDF or Word document instead. A network is protected by a [[Firewall]], so the attacker uses a USB drive to physically insert the malware. A voice analyst is unavailable, so the attacker calls pretending to be IT to reset credentials. Each vector exploits a different weakness in the human or technical environment—your job is to systematically harden *all* of them, not just the obvious ones.

## Exam Tips

- **Expect Scenario Questions**: The exam will present real-world attack scenarios and ask you to identify the vector (e.g., "An employee receives an email with a .docx attachment containing a macro that executes PowerShell commands. What vector is this?" Answer: **file-based vector** via **message-based delivery**).

- **Distinguish Between Vector and Payload**: A vector is the *method of delivery*; the payload is the *malware* itself. Don't confuse "[[Ransomware]]" (payload) with "USB flash drive" (vector). The exam tests whether you understand both.

- **File-Based Traps**: Remember that threat vectors aren't just executables (.exe). PDFs, Office documents, compressed archives, and even images can be weaponized. Many candidates overlook non-executable files as dangerous.

- **Message-Based Overlaps**: [[Phishing]] can use any channel (email, SMS, IM). [[Vishing]] is phishing via voice. **Smishing** is phishing via SMS. **Whaling** is phishing targeting executives. Know the terminology—the exam distinguishes them.

- **Removable Media is Critical**: In a homelab or enterprise setting, USB-based threats (including USB Rubber Ducky-style keyboard injection) are *real* and often underestimated. The exam emphasizes air-gapped systems vulnerable to physical vectors—expect 1–2 questions on this.

- **Defense-in-Depth Strategy**: The exam rewards candidates who think holistically. For email (message-based), you'd use [[SIEM]] filtering, [[IDS]]/[[IPS]], and attachment sandboxing. For USB (removable media), you'd use device control policies and [[Active Directory]] Group Policy. Show you understand *layered* defense.

---

## Common Mistakes

- **Treating All File Types the Same**: Candidates often assume only .exe files are dangerous. In reality, PDFs, Office macros, and SVG images are common attack vehicles. The exam tests whether you understand that threat vectors are broader than "executables."

- **Confusing Vector with Exploit**: A vector is the delivery method; an exploit is the code that triggers the vulnerability. You might deliver an exploit via email (vector), but the exam sometimes tests whether you're precise with language. Don't say "the email exploited the system"—say "the email *delivered* the exploit."

- **Overlooking Voice and Physical Vectors**: Many candidates focus exclusively on network-based threats (email, web) and underestimate vishing, war dialing, and USB attacks. These vectors are less glamorous but appear on the exam and are critical in real security operations (especially in critical infrastructure).

- **Not Considering Input Validation for Images**: The SVG/image vector is less obvious than email phishing. Candidates often miss that browsers must validate image content to prevent HTML/JavaScript injection. This is an easy "gotcha" question.

## Real-World Application

In your homelab, threat vectors translate directly to tactical defenses:

- **Message-Based**: Configure [[Wazuh]] to alert on suspicious email attachments (PDFs, Office docs with macros). Integrate with [[Active Directory]] to enforce [[MFA]] so phishing credentials alone don't grant access.

- **Removable Media**: Use [[Active Directory]] Group Policy to disable USB mass storage devices on workstations, or enforce mandatory encryption for USB devices. Monitor [[Wazuh]] for USB insertion events and file copies to USB drives.

- **Voice Vectors**: Implement inbound call filtering on VoIP systems; educate staff on vishing through security awareness training (validated in your SOC or [[SIEM]]).

- **File-Based in Homelab**: Test Office macro-based payloads in an isolated sandbox (e.g., using [[Metasploit]] or [[Kali Linux]]) to validate that your email [[IPS]] blocks them. Monitor [[Wazuh]] for suspicious process spawning from Office applications.

- **Air-Gapped Systems**: If running industrial control systems or classified networks in your lab, enforce strict USB policies and audit physical access—this is where USB-based threats become most dangerous.

**Before moving on, ensure you can:**

1. Define a threat vector and distinguish it from an exploit or payload.
2. Name at least 3 file types commonly used as threat vectors (PDF, Office, SVG, ZIP) and explain why each is dangerous.
3. Explain the difference between phishing, vishing, smishing, and whaling.
4. Describe how a USB device can infect an air-gapped system and why this is a critical threat.
