# SHA-512

## What it is
Like a meat grinder that turns any document — whether a haiku or a novel — into exactly 128 hex characters of indistinguishable hamburger, SHA-512 is a one-way cryptographic hash function from the SHA-2 family that produces a fixed 512-bit (64-byte) digest. No matter the input size, the output is always the same length, and reversing it is computationally infeasible.

## Why it matters
When a password database is breached, the difference between SHA-512-hashed passwords (especially with salting) and weaker hashes like MD5 is the difference between years of cracking time and minutes. Attackers using GPU-accelerated tools like Hashcat can crack unsalted MD5 hashes at billions of attempts per second, while a properly salted SHA-512 implementation dramatically raises that cost — this is why modern systems like Linux `/etc/shadow` use SHA-512 with salting by default.

## Key facts
- Produces a **512-bit (128 hex character)** digest; part of the SHA-2 family alongside SHA-256, SHA-384, and SHA-224
- **One-way function**: computationally infeasible to reverse; collision resistance is its primary security property
- SHA-512 processes data in **1024-bit blocks** using 80 rounds of compression — compared to SHA-256's 512-bit blocks and 64 rounds
- **Not a password storage solution alone** — must be paired with a salt (random value) and ideally a dedicated KDF like bcrypt or PBKDF2 to resist brute-force attacks
- SHA-1 and MD5 are considered **cryptographically broken** (collision attacks demonstrated); SHA-2 family including SHA-512 remains approved by NIST for use in federal systems
