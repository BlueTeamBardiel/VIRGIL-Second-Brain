# T1547.001 Registry Run Keys Persistence

## What it is
Think of the Windows Registry Run keys like an apartment building's morning announcement system — anything listed there gets automatically "announced" (executed) every time a user logs in. Precisely, these are specific Windows Registry locations (e.g., `HKCU\Software\Microsoft\Windows\CurrentVersion\Run`) where values pointing to executables are automatically launched at user logon or system startup, requiring no scheduled task or service installation.

## Why it matters
During the 2020 SolarWinds compromise, malware families used Run key entries to maintain persistence across reboots, ensuring the backdoor survived system restarts without relying on noisier mechanisms like new services. Defenders hunting this technique query the Run keys using tools like Autoruns (Sysinternals) or SIEM rules watching for `reg add` commands targeting these paths.

## Key facts
- **Primary locations**: `HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run` (all users) and `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` (current user only — requires lower privileges)
- **RunOnce variants** (`\RunOnce`) execute the entry a single time then delete it, often used by installers or malware evading persistent detection
- **HKLM keys require admin/SYSTEM privileges** to write; HKCU keys can be written by any standard user, making them a low-barrier persistence option
- **Detection telemetry**: Windows Event ID 13 (Registry value set) from Sysmon and `reg.exe` command-line auditing are primary detection signals
- **Mitigations** include application whitelisting (AppLocker/WDAC), monitoring Autoruns changes, and restricting registry write permissions via Group Policy
