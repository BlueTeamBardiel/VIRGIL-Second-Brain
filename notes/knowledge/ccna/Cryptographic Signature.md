# Cryptographic Signature

## What it is
Like a wax seal on a medieval letter — it proves both who sent it and that nobody tampered with the contents in transit. Precisely: a cryptographic signature is a mathematical value generated by hashing a message and encrypting that hash with the sender's **private key**, allowing anyone with the sender's **public key** to verify authenticity and integrity.

## Why it matters
In 2020, attackers behind the SolarWinds supply chain attack injected malicious code into software updates that were **legitimately signed** by SolarWinds' own compromised build server — demonstrating that signature validity only proves the key was used, not that the signing environment was trustworthy. This attack infected ~18,000 organizations including US federal agencies. Valid signatures became a false comfort, exposing the limits of code-signing trust chains.

## Key facts
- Cryptographic signatures provide **authentication** and **non-repudiation** — the signer cannot later deny signing without claiming their private key was compromised
- Common algorithms: **RSA**, **DSA**, **ECDSA** (Elliptic Curve, used in modern TLS and Bitcoin); ECDSA uses smaller keys for equivalent strength
- The message itself is **not encrypted** by signing — confidentiality is a separate concern requiring encryption
- Signature process: Hash(message) → encrypt hash with private key = signature; Verification: decrypt signature with public key → compare to Hash(message)
- **Certificate Authorities (CAs)** sign digital certificates to bind public keys to identities; a compromised CA (e.g., DigiNotar 2011) breaks the entire trust chain
- Hash algorithm weakness undermines signatures — MD5 collisions were exploited to forge CA certificates (Flame malware, 2012)
