## Universal Findings (all 4 sources agreed)

### 🔴 Critical
- Sanitization leak: private lab name and private paths in public repo
- curl|bash install pattern — RCE risk for security-aware audience
- Crontab as secrets store — eval on user-controlled input
- url-ingest.sh: no URL validation, SSRF vector, pandoc RCE history

### 🟠 High
- Unpinned Python dependencies (requests, feedparser, anthropic)
- No rate limiting on API calls — surprise billing risk
- No schema validation on Claude API responses in promote-patch.py
- triage-inbox.sh: prompt injection via malicious inbox note possible

### 🟡 Medium
- README leads with philosophy not value prop — conversion killer
- Empty vault on day 1 — no starter content
- Silent failures — missing API key = cryptic error
- No virgil status command — no way to see system health
- Mobile gap — no capture without SSH
- 22 RSS feeds on day 1 — overwhelming, no minimal mode
- Slash commands not discoverable in README

### 🟢 Low / Informational
- No unit tests or CI
- No undo for promote.sh memory writes
- Log files grow unbounded
- No multi-vault support
- Mac cron setup will surprise users (should use launchd)
